# Technology Radar M-OS — Adopt / Trial / Assess / Hold

> **Владелец:** innovation-director
> **Обновляется:** ежемесячно и после каждого market-scan
> **Последнее обновление:** 2026-04-18 (после Holding ERP Market Scan)
> **Методология:** заимствована у ThoughtWorks Technology Radar. 4 кольца:
> - **Adopt** — применять в M-OS сейчас, рекомендовано как стандарт
> - **Trial** — пробовать на ограниченном pilot'е, готовиться к Adopt
> - **Assess** — наблюдать и изучать, решение об пилоте — через 3–6 мес
> - **Hold** — не применять, есть известные проблемы

Каждая технология — с фазой (M-OS-1..4) и обоснованием.

---

## 1. ADOPT (применять сейчас)

| # | Технология | Где в M-OS | Обоснование |
|---|---|---|---|
| A1 | **Anti-corruption layer pattern** (DDD) | M-OS-1 | Необходимость, без него интеграции с внешним миром «размажутся» по ядру. ADR 0008/0009 уже зафиксировали |
| A2 | **Configuration-as-data** | M-OS-1 | Принцип 10 из vision. Все бизнес-сущности — в БД, не в коде. Без этого не выдержим требования Владельца «любой момент добавить/поменять/убрать» |
| A3 | **Pod-архитектура + Event Bus** | M-OS-1 | ADR 0009. Проверяется через добавление АЗС-pod-lite без переделки ядра |
| A4 | **Криптографическая цепочка хэшей аудита** | M-OS-1 | Уникальная функция M-OS, нет у конкурентов в РФ. Принцип 1 vision |
| A5 | **Telegram as primary notification + action channel** | M-OS-1 | Сотрудники холдинга уже в Telegram. Уникальность на российском рынке |
| A6 | **PWA + offline-first для field** | M-OS-1 | Стройка + АЗС — слабый интернет. PWA дешевле нативных apps |
| A7 | **Alembic + PostgreSQL round-trip migrations** | уже | ADR 0013 внедрён |
| A8 | **FastAPI + Pydantic** | уже | Стек Фаз 0–3 |
| A9 | **Fine-grained RBAC: роль + объект + действие** | M-OS-1 | Без этого 152-ФЗ не закроем. Принцип 6 vision |

## 4. HOLD (не применять)

| # | Технология | Почему Hold |
|---|---|---|
| H1 | **SAP / Oracle / Microsoft коммерческие ERP** | Недоступны юридически в РФ с 2022–2024. Не рассматриваем. Только как архитектурные референсы |
| H2 | **Нативные iOS/Android приложения вместо PWA (на данный момент)** | Vision §12: «начинаем с PWA». Нативные — позже, если потребуется. Нет ресурсов на 2 кодовые базы |
| H3 | **Прямой SQL-доступ к базам 1С** | Ломается на каждой миграции конфигурации. Только OData/REST |
| H4 | **Хранение ПД вне РФ** | ФЗ-152 + оборотные штрафы 500 млн. Строго запрещено |
| H5 | **Blockchain-based external audit** | Vision §12: рассмотрим после M-OS-2, скорее всего не требуется. Избыточная сложность |
| H6 | **Кастомизация SAP/Oracle «под себя»** | Birmingham, Lidl, Revlon уроки. Даже если бы SAP был доступен — «adopt not adapt», не наш путь |
| H7 | **Полная замена 1С с первого дня** | Vision §12: переход постепенный. Forced replacement ломает бухгалтерию |
| H8 | **Мультитенантный SaaS для внешних клиентов** | Out of scope до M-OS-5. White-label — возможно, после валидации внутри |
| H9 | **Live-интеграции без production-gate** | Feedback rules (2026-04-17): банки/Росреестр/1С/ОФД — только адаптеры на полке до production-gate. Исключение — Telegram |

## Правила ведения radar'а

1. Каждая новая технология добавляется либо сразу в Assess, либо (если есть внешний сильный сигнал + наше обоснование) сразу в Trial
2. Перевод из Trial → Adopt требует подтверждённого пилота в M-OS и ревью governance-director (через Координатора)
3. Перевод в Hold требует указания причины и ссылки на ADR/РФ-законодательство/finding
4. Radar ревьюется ежемесячно innovation-director и после каждого market-scan
