## Что меняется

### 1. Новый документ: `docs/agents/regulations_addendum_v1.6.md`

Формализует паттерн «Координатор-транспорт» для реализации иерархической делегации в условиях Claude Code, где субагенты не могут вызывать других субагентов.

Ключевое содержание:
- §1 Обоснование — инцидент ri-director + техническое ограничение Anthropic
- §2 Паттерн Координатор-транспорт — 8 стадий одной задачи
- §3 Правила для Координатора — никогда «сделай», дословный транзит
- §4 Правила для Директора — обязательный формат брифа, защитный отказ
- §5 Правила для Начальника отдела — формат плана распределения
- §6 Правила для Сотрудника — FILES_ALLOWED как граница
- §7 Техника Agent-вызовов
- §8 Исключения (Советники, XS, CLAUDE.md быстрый путь)

## Почему

Прямые цитаты Владельца (Telegram 2026-04-16):
- msg 733: «надо это исправлять чтобы директор не искал сам а лишь поручал поиск сотрудникам а потом проверял»
- msg 738: «а почему ты раньше не сказал что один субагент не может вызвать другого ты ошибаешься решай этот вопрос чтобы все работало как я задумал»
- msg 748: «останови managed и реализовывай путь 1 и начинай писать новый регламент»

Техническое обоснование:
- Claude Code sub-agents doc (https://code.claude.com/docs/en/sub-agents): «This prevents infinite nesting (subagents cannot spawn other subagents)»
- Managed Agents API multi-agent (https://platform.claude.com/docs/en/managed-agents/multi-agent): «Only one level of delegation is supported: the coordinator can call other agents, but those agents cannot call agents of their own»

Бизнес-обоснование:
- Иерархия ответственности (кто за что отвечает) сохраняется
- Физическая реализация приводится в соответствие с платформой Claude Code
- Предотвращает повторение инцидента «Директор делает работу Сотрудника»
- Даёт Координатору чёткие шаблоны, убирает неоднозначность
