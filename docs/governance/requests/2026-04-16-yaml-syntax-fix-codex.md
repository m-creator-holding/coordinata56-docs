## Что меняется

Три файла кодекса не парсились валидным YAML-загрузчиком. Внешний аудит (ChatGPT Codex) прогнал `yaml.safe_load(...)` и поймал три ошибки:

```
[FAIL] docs/agents/delegation-rules.yaml      → ParserError: expected <block end>, found '-' (line 33)
[FAIL] docs/agents/agents-map.yaml            → ParserError: expected <block end>, found '-' (line 26)
[FAIL] docs/agents/task-event-log.schema.yaml → ScannerError: mapping values not allowed (line 25)
```

### 1. `docs/agents/delegation-rules.yaml` (+553 / −552)

Список из 50 агентов шёл на корневом уровне сразу после `meta:`. По грамматике YAML нельзя смешивать mapping (`meta: {...}`) и sequence (`- id: ...`) в одном документе без ключа. Исправление — обернуть список под ключ `agents:` и сдвинуть indent +2:

```yaml
meta:
  version: "1.6"
  ...
agents:
  - id: owner
    level: L0
    ...
```

### 2. `docs/agents/agents-map.yaml` (+714 / −713)

Та же структурная ошибка: 55 паспортов агентов шли на корневом уровне. Обёрнуты под `agents:`, indent +2.

Дополнительно — закавычена одна строка, где двоеточие внутри значения ломало парсинг:
```diff
-  purpose: Формат: зачем → аналогия → конкретика → риски; никогда не тормозит проект
+  purpose: "Формат: зачем → аналогия → конкретика → риски; никогда не тормозит проект"
```

### 3. `docs/agents/task-event-log.schema.yaml` (+2 / −2)

Две строки с незакавыченным значением, содержащим `:` (Формат: ... и Коротко: ...) — закавычены:
```diff
-    description: Уникальный идентификатор события. Формат: `evt_<timestamp>_<random>`
+    description: "Уникальный идентификатор события. Формат: `evt_<timestamp>_<random>`"
-        description: Коротко: зачем делегация
+        description: "Коротко: зачем делегация"
```

## Почему — багфикс, а не изменение регламента

Правка чисто синтаксическая: делает файлы валидными для `yaml.safe_load`. До правки любой автоматический потребитель кодекса (будущий валидатор, дашборд, CI-проверка, скрипт обучения субагента на паспортах) падал на парсинге. Это дефект, не архитектурное решение.

Регламент — то, что читают агенты (люди и Claude) — **не меняется**: структура, поля, значения, комментарии идентичны. Меняется только способ упаковки в YAML-документ.
