## Что меняется

### 1. Отмена прямых маршрутов в v1.4 §5

**До:**
| Tier | Размер | Путь |
|---|---|---|
| XS | 1 файл, <1ч | Координатор → Worker напрямую |
| S | 2–10 файлов, один отдел | Координатор → Head → Worker |
| M | модуль | Координатор → Директор → Head → Workers |
| L | фаза | Координатор → несколько Директоров → ... |

**После:**
| Tier | Размер | Путь |
|---|---|---|
| XS | 1 файл, <1ч | Координатор → Директор → Worker (Директор может лично курировать при одном Worker'е в направлении) |
| S | 2–10 файлов, один отдел | Координатор → Директор → Head → Worker |
| M | модуль | Координатор → Директор → Head → Workers |
| L | фаза | Координатор → несколько Директоров → Heads → Workers |

Строгая цепочка: **Президент → Директор → Начальник → Сотрудник**, без сокращений по вертикали.

### 2. Исключение для Советников

Штабные Советники (architect, analyst, legal, construction-expert, tech-writer, memory-keeper, tutor, data-analyst) — **вне иерархии**. Координатор может обращаться к ним напрямую: это консультация, не задача. Данное исключение фиксируется в amendment явно.

### 3. Активация dormant-Директоров

Поскольку под «спящими» Директорами работают активные Сотрудники (`frontend-dev`, `designer`, `devops`, `db-engineer`, `integrator`), которые по новому правилу не могут получать задачи через голову Директора, необходимо активировать:
- `frontend-director` — active-супервайзер для `frontend-dev`
- `design-director` — active-супервайзер для `designer`
- `infra-director` — active-супервайзер для `devops`, `db-engineer`, `integrator`
- `legal-director` — **остаётся dormant** (юр-вопросы закрывает Советник `legal`, который вне иерархии)

Начальники отделов (integrator-head, frontend-head, ux-head, visual-head, content-head, devops-head, db-head, legal-head) — **остаются dormant** до момента, когда под ними появится >1 Worker. Директор курирует Worker напрямую при одиночном Worker'е.

## Почему

Прямая цитата Владельца (Telegram msg 665, 2026-04-16):
> «кординатор не может дать задачу сотруднику кординатор ставит задачу директору департамента дальше директор ставит задачу начальнику отдела только потом сотрудник получает. ты президент ты можешь ставить задачу только директорам»

Бизнес-обоснование (Координатор):
1. Единый канал ответственности — Директор всегда знает, что происходит в его направлении
2. Нет «микроменеджмента Президента» — роли разделены
3. Защита от того, что Координатор нагружает Worker'а без ведома Директора
4. Согласуется с принципом CEO-модели, на которую Мартин ссылается в своём холдинге
