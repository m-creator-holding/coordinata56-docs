## Описание

Внешний аудитор обнаружил в исходном коде прототипа три нарушения CODE_OF_LAWS v2.1 ст. 45а («запрет живых внешних интеграций до production-gate»):

| # | Нарушение | Файл | Статья |
|---|---|---|---|

| 3 | Hardcoded API key в коде | в тех же файлах | ст. 45а (три обязательных столпа), CLAUDE.md §«Секреты и тесты» |

Все три нарушения:
- находятся вне `integration_catalog` (реестра не существует ещё),
- без ACL-обёртки (`IntegrationAdapter` каркас не реализован),
- без состояния адаптера (`written` / `enabled_mock` / `enabled_live`),
- содержат чувствительные данные (hardcoded credentials).

**Дополнительный контекст:** на момент фиксации инцидента прототип существует в стеке Next.js + Supabase, который сам по себе требует решения Владельца о сохранении или возврате к Foundation (backend/Alembic/Vite). Технический стек-вопрос выходит за границы governance — это бизнес-решение архитектурного уровня, эскалируемое Координатором.

## Нормативная база

**CODE_OF_LAWS v2.1 ст. 45а** (дословно):
> Запрет живых внешних интеграций до production-gate. Три обязательных столпа (юрид. + security-аудит + staging симуляции) должны быть пройдены перед активацией любой внешней интеграции, кроме Telegram. Активация без прохождения всех столпов — нарушение.

**CLAUDE.md** раздел «Секреты и тесты»:
> «Никогда не литералить пароли, токены, секреты — ни в src/, ни в tests/, ни в conftest.py»

## Немедленные действия (порядок, не блокирует остальную работу)

1. **Ротация Anthropic API key** — выполняет Владелец (только у него доступ к console.anthropic.com). Приоритет: немедленно после прочтения доклада. Это действие не зависит от технического решения по стеку.
2. **Ротация Supabase credentials** (если использовались service_role key или anon key с расширенными правами) — выполняет Владелец.
3. **Очистка кода от hardcoded secrets** — выполняет Координатор через backend-director после решения Владельца по стеку (Next.js остаётся или возвращаемся к Foundation). До решения по стеку чистить бессмысленно — файлы могут быть удалены целиком.

## Что НЕ делать без решения Координатора

- Не делать экстренных коммитов в прототип: стек под вопросом, коммит может зафиксировать неверный путь.
- Не закрывать /api/* без auth guard до решения по стеку: один и тот же риск, одно решение.
- Не подключать никаких других внешних API до прохождения production-gate (три столпа, ст. 45а).
