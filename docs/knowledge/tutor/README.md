# Курс «Программирование через M-OS»

Персональный обучающий курс для Мартина — Владельца проекта `coordinata56`.

**Принцип**: обучение встроено в проект. Каждый урок объясняет концепцию программирования на реальном коде M-OS — не выдуманных примерах. Теория — только там, где без неё нельзя понять практику.

**Формат каждого урока**: 15-20 минут чтения. Зачем → Аналогия из бизнеса/стройки → Реальный код проекта → Как объяснить на совещании → Риски → Проверка понимания.

## Рекомендуемый темп

**Минимальный**: 1 урок в день = 15-20 минут утром перед работой.

**Оптимальный**: 2-3 урока в неделю, когда есть свежая задача от команды по теме урока.

**Не рекомендуется**: читать более 2 уроков подряд за один раз — информация не усваивается.

---

## Как использовать курс

1. **Перед чтением** — откройте файл кода, упомянутый в уроке, в любом текстовом редакторе или на GitHub. Читать урок и код параллельно.

2. **Во время чтения** — если что-то непонятно, пропустите и читайте дальше. Концепция прояснится в следующем уроке или при чтении другого файла проекта.

3. **После урока** — ответьте на вопросы из раздела «Проверка понимания» мысленно. Если ответа нет — перечитайте соответствующий раздел.

4. **На следующий день** — попробуйте объяснить тему урока своими словами (в формате «как бы Вы рассказали на совещании»). Если получается — материал усвоен.

## Структура файлов проекта (навигатор)

```
coordinata56/
  backend/
    app/
      api/          ← HTTP-эндпоинты (урок 06)
        health.py   ← простейший пример API
      models/       ← описание таблиц БД (урок 08)
        user.py     ← модель пользователя
        audit.py    ← модель журнала аудита
      services/     ← бизнес-логика (урок 09)
        audit.py    ← сервис аудита
        budget_plan.py ← сервис плана бюджета
      repositories/ ← работа с базой данных (урок 07)
        base.py     ← базовый репозиторий
      schemas/      ← контракты данных (валидация)
        auth.py     ← схемы для логина
      db/
        session.py  ← подключение к базе данных
    alembic/
      versions/     ← миграции базы данных (урок 10)
  frontend/
    src/
      pages/        ← страницы интерфейса (урок 14)
        DashboardPage.tsx
      components/   ← переиспользуемые блоки UI
  docs/
    adr/            ← архитектурные решения
      0002-tech-stack.md ← почему Python, React (урок 02)
    knowledge/
      tutor/        ← вы здесь
      glossary.md   ← глоссарий терминов
```
