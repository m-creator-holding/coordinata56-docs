# Best practices для застройщика коттеджей (бизнес-класс)

**Документ:** 05-best-practices.md
**Версия:** 1.0
**Дата:** 2026-04-17
**Автор:** construction-expert (Советник L4)
**Аудитория:** Владелец, директор проекта, коммерческий директор, новые сотрудники
**Применение:** оперативные практики для ежедневной работы; чек-листы критических решений

> **Логика документа.** Каждый раздел — конкретная операционная практика: что проверять, как принимать решения, на что смотреть. Это не общие советы, а конкретные действия, которые должны войти в регламенты, BPM и чек-листы системы.

## Раздел 1. Как выбирать подрядчиков

### Правило 1.1. Не берите подрядчика «с улицы»
**Проблема:** самый дешёвый подрядчик часто срывает сроки, делает брак, или пропадает с авансом. Экономия 10% на этапе превращается в 30–50% переделок.

**Что делать:**
1. Искать подрядчиков через три источника параллельно:
   - Рекомендации от проверенных коллег (прораб, ГИП, партнёры по другим стройкам).
   - Профильные ассоциации и СРО — они дают реестры членов с рейтингами.
   - Объективные площадки: «ЕИС закупки», «Строительный мост», Яндекс.Услуги с отзывами.
2. Никогда не заключать первый договор на сумму более 2–3 млн ₽. Первый договор = тест.
3. На тестовом договоре проверять: сроки, качество, коммуникацию, юридическую дисциплину.

### Правило 1.2. Чек-лист проверки подрядчика перед подписанием договора

Перед подписанием любого договора с суммой от 500 тыс. ₽ проверяем:

| № | Пункт | Где проверить | Что должно быть |
|---|---|---|---|
| 1 | Наличие юрлица / ИП в ЕГРЮЛ/ЕГРИП | egrul.nalog.ru | Активен, не в ликвидации |
| 2 | Возраст компании | там же | ≥ 2 лет для крупных работ |
| 3 | ИНН, ОГРН, юридический адрес | договор, выписка | Совпадают с реестром |
| 4 | Членство в СРО (для работ от 3 млн ₽) | reestr-sro.ru | Действующее, с нужным видом работ |
| 5 | Лицензии (если требуются — электрика высокого напряжения, газ) | lic.minstroyrf.ru | Действующие |
| 6 | Отсутствие банкротства | bankrot.fedresurs.ru | Нет процедур |
| 7 | Отсутствие исполнительных производств | fssp.gov.ru | Нет долгов свыше 1 млн ₽ |
| 8 | Судебные дела | kad.arbitr.ru | Нет массовых исков от заказчиков |
| 9 | Налоговая дисциплина | pb.nalog.ru (прозрачный бизнес) | Нет налоговых долгов |
| 10 | Рекомендации | запросить 3 контакта + обзвонить | Минимум 2 положительных отзыва |
| 11 | Портфолио — реальные фото объектов | сайт, соцсети | Есть свежие работы (2024–2025) |
| 12 | Выезд на действующий объект | договориться | Лично увидеть, как они работают |

**Красные флаги (работать нельзя):**
- Компания младше 1 года без внятной предыстории команды.
- Нет офиса, только мобильник.
- Отказываются показать действующий объект.
- Настаивают на оплате только наличными или на ИП директора, а не на юрлицо.
- В судах массовые иски от заказчиков.
- Юрадрес = «массовая регистрация».

### Правило 1.3. Платить только по факту
**Никогда** не платить 100% аванса. Норма:
- Аванс 20–30% на мобилизацию (материалы, техника, первая зарплата).
- Поэтапные платежи по подписанным КС-2 / КС-3, каждый раз за конкретный объём.
- Финальный платёж 10% — через 30–60 дней после сдачи, после устранения всех замечаний.

## Раздел 2. Как вести план/факт по бюджету

### Правило 2.1. Плановый бюджет — до начала работ, не после
**Проблема:** многие малые застройщики «бюджет уточняют по ходу». Итог: в конце проекта сюрприз на минус 15–30%.

**Что делать:**
1. До подписания первого договора — собрать **сводный сметный расчёт (ССР)** со всеми статьями, включая непредвиденные (резерв 5–10%).
2. Разбить ССР по стадиям стройки и по статьям бюджета.
3. Ввести в систему coordinata56 в `budget_plan` (разрез: проект × статья × стадия × дом).
4. Любое изменение — через **журнал изменений бюджета** с обоснованием.

### Правило 2.2. Фактические платежи — только с привязкой к договору и статье
**Проблема:** «висящие» платежи без документального основания — это путь к воровству и потере контроля.

**Что делать:**
1. Каждый платёж в системе — с обязательной привязкой к `contract_id` и `budget_category_id`.
2. Без привязки система не даёт создать платёж (это уже реализовано в coordinata56 через поле Q2 из KPI).
3. Если платёж «срочный» без договора — это исключение, которое утверждает только Владелец.

### Правило 2.3. Еженедельный план/факт — разбор, а не просто отчёт
**Что делать:**
- Каждый понедельник — 30 минут с директором проекта: смотрим отклонения.
- По каждому отклонению >3% — «почему?».
- Фиксируем корректирующие действия, ставим дедлайн.
- Через неделю проверяем, что сделано.

### Правило 2.4. Закрытие бюджета по стадии — обязательное
Когда стадия закрывается:
1. Сверяем план / факт по всем статьям этой стадии.
2. Если экономия — перераспределяем в резерв проекта (не в следующую стадию).
3. Если перерасход — делаем обоснование и утверждаем у Владельца.
4. Закрываем стадию в системе, она становится read-only.

## Раздел 3. Как контролировать качество

### Правило 3.1. Входной контроль материалов — без компромиссов
**Проблема:** 30% проблем на стройке — плохие материалы. «Серые» цементные мешки, блоки не той марки, арматура не по ГОСТ.

**Что делать:**
Каждая поставка материала на объект проходит проверку:
1. **Паспорт качества** — обязательно. Нет паспорта — не принимаем.
2. **Марка / класс** совпадает с заявленным в заказе. Например, бетон B25, а не B20.
3. **Внешний осмотр**: блоки не битые, арматура не ржавая, мешки цемента целые и не слежавшиеся.
4. **Выборочный лабораторный контроль** для критичных позиций: бетон — контрольный кубик, арматура — механические испытания (для крупных объектов).
5. **Регистрация в журнале входного контроля** с подписью ответственного.

**Отказ от партии** оформляется актом, поставщик забирает материал за свой счёт.

### Правило 3.2. Промежуточные приёмки — каждую стадию
**Что делать:**
- Каждый АОСР проверяет **тройка**: представитель подрядчика + технадзор заказчика + авторский надзор.
- Без трёх подписей АОСР не считается закрытым.
- Следующий этап не начинается до закрытия АОСР предыдущего (это можно и нужно автоматизировать в BPM).

### Правило 3.3. Внешний технадзор — платим, не экономим
**Проблема:** свой технадзор может быть «в доле» с прорабом подрядчика. Внешний — независимый.

**Что делать:**
- На объекты от 20 млн ₽ — обязательно нанимать внешний технадзор.
- Стоимость: 1,5–3% от сметной стоимости объекта.
- Отчёт технадзора — еженедельный, письменный, с фотофиксацией.

### Правило 3.4. Фотофиксация — правило «20 фото на стадию»
**Что делать:**
- На каждую стадию каждого дома — минимум 20 фотографий.
- Каждое фото с датой, геолокацией, привязкой к объекту.
- Хранится в системе coordinata56 с привязкой к `house_id` и `stage_id`.
- Особенно тщательно — скрытые работы (арматура перед заливкой, гидроизоляция перед засыпкой).

### Правило 3.5. Тепловизор и влагомер — обязательный инвентарь
**Что делать:**
- После утепления — тепловизорная съёмка. Мостики холода — на фото.
- Перед чистовой отделкой стен — замер влажности штукатурки / стяжки. Норма: не выше 4% влажности.
- Эти данные — в карточку дома в системе.

## Раздел 4. Как работать с клиентами (CRM для покупателей коттеджей)

### Критические точки коммуникации

Покупатель коттеджа проходит **8 ключевых контактов** с застройщиком. Каждый — точка, где можно потерять или закрепить клиента.

#### Точка 1. Первичный контакт (заявка)
**Что важно:**
- Ответ на заявку — **в течение 15 минут в рабочее время**. Каждые лишние 15 минут — минус ~10% конверсии.
- Первый контакт — живой человек, не бот.
- Задача первого контакта: квалификация. Понять — целевой клиент или нет.

**Что делать:**
- Система coordinata56 автоматически уведомляет отдел продаж.
- Менеджер созванивается, заполняет квалификационную форму: бюджет, сроки, сейчас живёт в, мотивация покупки.
- Квалифицированный клиент → следующая точка в течение 2–3 дней.

#### Точка 2. Показ объекта
**Что важно:**
- Показывают **построенные** или хотя бы активно строящиеся дома, а не «рендеры и котлован».
- Показывает менеджер, знающий технические детали (материалы, инженерия, этапы).
- Обязательно — территория посёлка, общая инфраструктура.

**Что делать:**
- В системе — расписание показов с закреплёнными менеджерами.
- После показа — карточка визита с заметками менеджера и оценкой клиента («горячий / тёплый / холодный»).

#### Точка 3. Бронирование
**Что важно:**
- Бронь — это платное обязательство, а не «отложим дом на слово».
- Типовая сумма бронирования для бизнес-класса: 100–500 тыс. ₽, возвратная при отказе от сделки в 14 дней.

**Что делать:**
- Договор бронирования — стандартная форма, в системе.
- После оплаты — дом в статусе «забронирован», не показывается другим клиентам в течение брони.

#### Точка 4. Согласование опций (конфигуратор)
**Что важно:**
- Покупатель выбирает: тип дома (A/B/C/D), опции (терраса, спа, интерьерный пакет, ландшафт). Цена фиксируется на момент выбора.
- Нужно дать клиенту время и инструменты: визуализацию, смету, консультацию.

**Что делать:**
- В «Координата 56» модуль конфигуратора планируется на M-OS-4.
- До этого — согласование через встречи + таблицы + договор.
- Ключевой документ: **спецификация дома** с подписью покупателя.

#### Точка 5. ДКП / ДДУ
**Что важно:**
- Договор — юридически чистый, прошедший проверку юриста.
- Клиент должен понимать условия (особенно ипотеку, эскроу, сроки сдачи).
- Нельзя перегружать договор оговорками в пользу застройщика — это отпугивает на этапе подписания.

**Что делать:**
- Шаблон договора — один, согласованный юристом холдинга.
- Любые исключения — только с согласования юриста.
- В системе — подписанный договор с отметкой о дате, сумме, сроках.

#### Точка 6. Строительство (гости на стройке)
**Что важно:**
- Клиент хочет видеть прогресс. Игнорирование = потеря доверия, расторжение, суды.
- Злоупотребление присутствием клиента = помехи бригадам.

**Что делать:**
- Еженедельный **отчёт о ходе работ** клиенту: 5–10 фото, текстовый комментарий, % готовности. Автоматически из системы.
- Допуск на стройку — по расписанию, с сопровождением менеджера. Не реже 1 раза в месяц.
- Видеотрансляция с объекта (камера) — хороший тон для бизнес-класса.

#### Точка 7. Предсдаточная приёмка
**Что важно:**
- Перед официальной сдачей — встреча с клиентом на объекте. Клиент проходит по дому, фиксирует замечания.
- **Все замечания** — устраняются до подписания акта передачи.

**Что делать:**
- В системе — лист приёмки с клиентом, замечания + сроки устранения.
- Контроль устранения — через чек-лист.

#### Точка 8. Передача ключей и гарантийный период
**Что важно:**
- Передача — торжественное событие. Клиент чувствует, что для вас это важно.
- В акте передачи — полный комплект документов для клиента: паспорта на оборудование, гарантии, инструкции, контакты сервиса.
- Гарантия на дом — **минимум 5 лет** (ГК РФ), на конструктивные элементы — 10 лет. На инженерное оборудование — по гарантиям производителя.

**Что делать:**
- Церемония передачи ключей, небольшой welcome-пакет.
- В системе — акт передачи, гарантийный талон.
- Назначение менеджера по сервису — **один контакт** для клиента на гарантийный период.

### Правило 4.1. Один менеджер ведёт клиента от точки 1 до точки 8
Переключения между менеджерами = потеря информации = негатив клиента. Если менеджер увольняется / в отпуске — передача дела с полной историей.

### Правило 4.2. CRM — это не excel-таблица
Для бизнес-класса минимум:
- Вся история звонков, встреч, переписки — в одном месте.
- Статус клиента по воронке.
- Автоматические напоминания менеджеру (перезвонить через N дней, отправить отчёт).
- Связь с данными стройки (клиент видит прогресс своего дома).

## Раздел 5. Типовые 10 ошибок малых девелоперов

Эти ошибки повторяют 80% молодых застройщиков. Зная их — можно не повторять.

### Ошибка 1. Нет детального бюджета до начала
Начали стройку «по оценкам», потом выяснили — не хватает 15–25%. Цена ошибки: кассовый разрыв, долги, стоп стройки на месяцы.

**Как избежать:** ССР до первого договора. Резерв на непредвиденные — 7–10%, не меньше.

### Ошибка 2. Экономия на изысканиях и проекте
Сэкономили 500 тыс. ₽ на геологии и полноценном проекте — потеряли 5–15 млн ₽ на переделке фундамента / инженерии. 10–15% случаев — дом под снос.

**Как избежать:** ИГИ + полный проект + авторский надзор — это священное. Не трогать, даже если «денег в обрез».

### Ошибка 3. Работа без договоров и актов
«Свои пацаны», «по-человечески», «без бумажек — быстрее». Результат: когда возникает спор — доказать ничего нельзя, деньги уходят, сроки срываются.

**Как избежать:** любой подрядчик — через договор. Любая работа — через КС-2. Любая скрытая работа — через АОСР.

### Ошибка 4. 100% предоплата подрядчику
Платят весь договор вперёд «для мотивации». Подрядчик пропадает или делает медленно, потому что деньги уже получил.

**Как избежать:** максимум 30% аванс, остальное — по факту. Никогда 100% вперёд.

### Ошибка 5. Отсутствие технадзора
«Сами контролируем». На практике «сами» — это прораб, которого подкупил подрядчик. Качество падает, никто не видит.

**Как избежать:** внешний технадзор от 1,5% сметы. Окупается многократно.

### Ошибка 6. Нет фотофиксации скрытых работ
Замуровали арматуру / гидроизоляцию / проводку и забыли сфотографировать. Через 5 лет возникла проблема — невозможно доказать, что сделали правильно.

**Как избежать:** строгое правило «не сфоткал — не подписал АОСР».

### Ошибка 7. Экономия на отделке МОП / фасаде
У коттеджа это = фасад и благоустройство. «Внутри сделаем хорошо, а снаружи потом». Клиент уже впечатлён негативно ещё до входа в дом.

**Как избежать:** для бизнес-класса фасад и благоустройство — это 15–25% бюджета. Не меньше. Это видно покупателю в первую очередь.

### Ошибка 8. Параллельные стройки без ритма
Начали все 85 домов одновременно — все на стадии фундамента, потом все на кладке. Пики нагрузки на один подрядчик, провалы когда «идём между стадиями».

**Как избежать:** ритмичный запуск: первая очередь 10–15 домов, через 2 месяца следующая очередь. Равномерный фронт.

### Ошибка 9. Игнорирование гарантийного периода
Передали ключи — «всё, забудьте». Потом клиент звонит с проблемой — никто не отвечает. Минус репутация, минус сарафан, минус следующие продажи.

**Как избежать:** выделенный сервис-менеджер на гарантийный период. Бюджет на гарантийные работы — 1–2% от выручки в резерве.

### Ошибка 10. Отсутствие учёта материалов по домам
Куплено тонн цемента много, где они — никто не знает. Часть украли, часть испортили, часть переиспользовали без учёта.

**Как избежать:** учёт материалов с привязкой к `house_id` и `stage_id`. Каждый мешок, каждый куб — на счёте конкретного дома.

## Раздел 6. Календарь регулярных обязательных действий

Что должно происходить регулярно, чтобы стройка шла без сюрпризов:

| Периодичность | Действие | Ответственный |
|---|---|---|
| **Ежедневно** | Запись в журнал работ | Прораб |
| **Ежедневно** | Просмотр остатка денежных средств | Финдиректор |
| **Ежедневно** | Обзвон «горячих» клиентов | Менеджер продаж |
| **Раз в 2–3 дня** | Посещение объекта директором проекта | Директор проекта |
| **Еженедельно** | Совещание «план/факт» бюджета | Владелец + финдиректор + директор проекта |
| **Еженедельно** | Отчёт клиенту по ходу стройки | Менеджер |
| **Еженедельно** | Проверка качества кладки / монолита / скрытых работ | Технадзор |
| **Раз в 2 недели** | Визит авторского надзора | ГИП |
| **Ежемесячно** | KPI-совещание | Владелец + все директора |
| **Ежемесячно** | Инвентаризация складов материалов | Снабженец + бухгалтер |
| **Ежемесячно** | Зарплата и акты с подрядчиками | Финдиректор |
| **Ежеквартально** | Ретроспектива качества (разбор дефектов и рекламаций) | ГИП + технадзор |
| **Ежеквартально** | Ревизия пула подрядчиков | Директор проекта |
| **Ежегодно** | Стратегическая сессия: что строим дальше, какие ошибки года | Владелец + правление |

## Раздел 7. Ключевые принципы (резюме)

1. **Бюджет — до начала работ**. Резерв 7–10%. Любое отклонение — через журнал изменений.
2. **Подрядчик — проверенный, с договором, без 100% аванса**.
3. **Качество — через входной контроль + АОСР с тройкой + внешний технадзор + фото**.
4. **Клиент — один менеджер, 8 точек контакта, еженедельный отчёт**.
5. **Документы — каждый день, каждую стадию, в системе, не в папках**.
6. **Скрытые работы — фотофиксация обязательна, иначе не засчитываются**.
7. **Ритм стройки — равномерный фронт, очередями, а не всё сразу**.
8. **Гарантия — не провалить: сервис-менеджер, бюджет, быстрая реакция**.
