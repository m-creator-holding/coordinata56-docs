## 1. Claude Code Routines

**Источник:** platform.claude.com/docs/en/release-notes/claude-code (Anthropic, 14.04.2026)
**Что это.** Автономный запуск Claude Code в облаке Anthropic по cron-расписанию, HTTP API или GitHub-webhook. 15 запусков/день в подписке. Обсуждается в RFC-001.

**Внедрение.**
- Не устанавливается локально — это feature в веб-интерфейсе Claude Code.
- У нас уже есть встроенный скил `schedule` (Claude Code ships with it), который создаёт scheduled remote agents. По описанию — это обёртка над Routines.

**Статус пилота.** Готов к запуску. Команда для запуска в следующей сессии:
```
Мартин в терминале: claude → /schedule create
  prompt: "Прочти docs/research/findings.md и docs/research/rfc/; собери 3-7 свежих находок R&I с расшифровкой простым русским; отправь в Telegram chat <OWNER_CHAT_ID> как weekly digest."
  cron: "0 10 * * MON"
  channel: plugin:telegram
```

**Рекомендация Координатора.** **Pilot — ДА.** Стоит 0 ₽, откат за минуту, закрывает живую боль (weekly digest теряется из-за моей памяти между сессиями).

**Блокер для запуска сейчас:** настройка `/schedule` требует интерактивной сессии Владельца в терминале. Я могу подготовить всё, но триггер физически создаётся из запущенного `claude` под Владельцем.

## 2. claude-mem

**Источник:** github.com/thedotmack/claude-mem (14.04.2026)
**Что это.** Плагин для Claude Code со сжатой внешней памятью между сессиями. `npx claude-mem install` устанавливает плагин.

**Внедрение.**
- Пакет `claude-mem@12.1.5`, лицензия **AGPL-3.0** (важно: при серверной эксплуатации требует раскрытия модификаций; это вопрос для `legal`).
- Размер 4.6 МБ, 13 зависимостей. Не малютка.
- Установка модифицирует `~/.claude/` — та же директория, где у нас свои 48 агентов и скилы.

**Статус пилота.** ⏸️ **НЕ УСТАНОВЛЕНО автоматически. Причины:**
1. **AGPL-3.0** — юридически требует обсуждения с `legal` (универсальный советник). При коммерческом развёртывании на сервере — disclosure. Для личного использования в MVP — приемлемо, но это решение Владельца.
2. **Дублирование с нашей системой памяти.** У нас уже есть `/root/.claude/projects/-root/memory/` с 20+ файлами, `memory-keeper` как хранитель, правило journaling tasks. claude-mem может конфликтовать или перекрывать.
3. **4.6 МБ + 13 зависимостей** внесут новые npm-пакеты в `~/.claude/`, трекинг лицензий усложнится.

**Рекомендация Координатора.** **SKIP / defer.** Основание:
- У нас 1M контекст Opus — не упираемся в 200K лимит
- Наша memory-система работает и управляема
- AGPL — отдельный юридический вопрос
- Эффективнее допилить memory-keeper под наши нужды, чем встраивать чужой плагин

Взять на заметку — если через 3–6 месяцев упрёмся в лимит или система памяти начнёт проседать, вернуться.

## 4. ccxray

**Источник:** github.com/lis186/ccxray (10.04.2026)
**Что это.** HTTP-прокси между Claude Code и Anthropic API с дашбордом: Timeline, Usage & Cost, System Prompt Tracking. Лицензия **MIT**, `ccxray@1.5.0`.

**Внедрение.**
- Устанавливается через `npx ccxray claude`
- Node.js 18+ (у нас есть), 2 зависимости (малый footprint)
- API-ключ проходит через прокси — требуется доверие к коду (MIT-open, проверяемо)

**Статус пилота.** ⏸️ **ОТЛОЖЕНО — не запущено сейчас.** Причины:
- `npx ccxray claude` запускает Claude Code через прокси, что прервёт текущую сессию (я сейчас в ней работаю на Вас)
- Дашборд ccxray открывает локальный веб-сервер — нужен браузер у Вас, а мы работаем через Telegram
- Для полноценного пилота нужна сессия вне Telegram-канала

**Рекомендация Координатора.** **Pilot при следующей большой сессии.** Когда будете работать через терминал — запустите `npx ccxray claude` вместо `claude`, увидите дашборд с живыми метриками: сколько токенов жёт каждый субагент, сколько стоит, какой system-prompt активен.

Практическая польза растёт при росте числа активных субагентов. С текущими 27 активными — приятно иметь, но не критично. С 40+ — становится важно для бюджет-контроля.

## 6. hermes-agent (NousResearch)

**Источник:** github.com/NousResearch/hermes-agent (14.04.2026)
**Что это.** Самостоятельная платформа AI-агентов от NousResearch с циклом обучения, встроенным cron-scheduler'ом, памятью, мультиплатформа (Telegram/Discord/Slack/WhatsApp).

**Внедрение.**
- Устанавливается через `curl | bash` (скрипт из GitHub)
- Работает с любыми LLM-провайдерами: OpenAI, Anthropic, OpenRouter, локальные
- Это НЕ плагин Claude Code. Это **альтернатива** Claude Code с собственным CLI `hermes`

**Статус пилота.** ❌ **REJECT — не устанавливаем.** Причины:
1. Hermes и Claude Code — **конкурирующие платформы**, не взаимодополняющие. Установить hermes = вести параллельную систему.
2. Мы уже выбрали Claude Code с Opus 4.6 и 1M контекста как единую платформу (подтверждено регламентом v1.4 §«Модели»: «остаёмся на линейке Claude до закрытия MVP»).
3. Cron-scheduler, мульти-канальная связь, память, субагенты — **всё это у нас уже есть** через Claude Code (schedule/loop скилы, Telegram-плагин, memory-keeper, 48 субагентов).
4. Добавление второй платформы = раздвоение памяти, раздвоение регламента, конфликт в `~/.claude/` (хотя hermes ставит в своё место).
5. Высокая звёздность — тренд, но не критерий совместимости с нашим стеком.

**Рекомендация Координатора.** Оставить как reference в findings.md, не пилотировать. Если в M4+ (маркетинг/продажи после MVP) появится потребность в агенте через Telegram/Discord без Claude Code — тогда вернуться.
