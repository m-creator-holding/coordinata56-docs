# Mine 2 — git add -A с 15 файлами и 3 «чужими»

**Тестируемый хук:** H-2 (pre-commit git add -A guard)

**Что симулируем:** Координатор (или любой исполнитель) сделал `git add -A` при 15 staged-файлах, из которых 3 — от «чужого» background-агента (mtime < 30 минут, в active-workers.json).

**Ожидаемая реакция:**
- `git commit` либо блокируется (автономный режим без TTY), либо выводит warning и требует подтверждения.
- stderr содержит substring `HOOK H-2` или `WARNING H-2`.
- stderr перечисляет «подозрительные файлы» — те 3, что помечены как «чужие».

**Критические требования к фикстуре (из §5.5 брифа):**
1. Эвристика H-2 «чужие файлы» зависит от `mtime` и реестра `.claude/active-workers.json`. Фикстура **явно** создаёт реестр с нужными путями перед запуском.
2. Файлы «чужого» агента должны иметь свежий mtime (touch на момент прогона, не заранее).
3. Без реестра — мина воспроизведётся недетерминированно.

**Артефакты:**
- `reproduce.sh` — создаёт 15 файлов в `/tmp/mine2-staging/`, 3 из них отмечает как «чужие» через reestr, делает `git add -A`, пытается commit.
- `active-workers-fixture.json` — шаблон реестра с указанием «чужих» файлов.

**Успех:**
- block или warn+substring. Exit code для warn допустим 0 (см. §5.2 брифа — warn=stderr substring, не exit code).
- В тексте warning/block упоминаются 3 подозрительных файла.

**Обходной путь H-2 (для чистого сценария clean-scenarios.md):**
- ENV `COORDINATOR_ALLOW_ADD_ALL=1` — разрешение разовое.
- `COMMIT_CONFIRMED=1` — интерактивное подтверждение симулировано через переменную.
