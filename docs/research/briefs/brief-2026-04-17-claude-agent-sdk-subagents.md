# Claude Agent SDK — официальная документация по субагентам — мини-бриф

**Slug:** claude-agent-sdk-subagents
**Дата:** 2026-04-17
**Источник:** https://www.anthropic.com/engineering/building-agents-with-the-claude-agent-sdk
**Теги:** claude-sdk, subagents, context-isolation, R&I-transport

## Что это делает

Официальная статья Anthropic объясняет как работают субагенты в Claude Agent SDK (наборе инструментов Anthropic для построения агентных систем). Ключевой факт: каждый субагент работает в своём изолированном «окне памяти» — он видит только свою задачу, не всю историю главного агента. Когда субагент заканчивает работу — он возвращает только итоговый результат, не промежуточные шаги. И официально подтверждено: субагент не может сам запустить другого субагента — только главный оркестратор.

## Чем полезно нам

Это официальное подтверждение текущего ограничения, описанного в нашем регламенте v1.6 (паттерн «Координатор-транспорт»). Статья также описывает правильную модель изоляции контекста — наш риск «контекстного загрязнения» снижается именно так: каждый агент видит только свой бриф.

## Что предлагаем

1. Цитировать эту статью в RFC-004 как официальное техническое обоснование ограничения.
2. Рекомендовать ri-analyst изучить раздел про «передавай только нужное» как основу для формата бриф-сообщений между Координатором и субагентами.
