## 3. Топ-3 для ближайшего внедрения

### 1. lychee (Adopt light) — проверка ссылок в `docs/`
**Что даёт coordinata56:** первый прогон сегодня покажет все битые ссылки между регламентами, ADR, департаментами. Закрывает Д-05 и У-06 из Governance-отчёта. Аудитор перестанет тратить 40% времени на ручной обход, сосредоточится на смысле. Одна команда, никакой инфраструктуры, без внешних сервисов.
**Трудозатраты:** 1 час Analyst'а на первый прогон + правка регламента аудита.

### 2. markdownlint + frontmatter-schema (Trial) — структура регламентов
**Что даёт coordinata56:** ловит Д-01 (дубли разделов), Д-03 (расходящиеся формулировки в 3 местах), П-04 (нет единого шаблона ADR) автоматически. Выявит системные расхождения в 14 ADR, которые сейчас написаны в разной структуре. Требует Node.js (уже есть), одна команда запуска.
**Трудозатраты:** 3 часа Analyst'а на ruleset + черновик JSON-схемы.

### 3. Mem0 (Trial, только self-hosted) — память агентов на M-OS-1+
**Что даёт coordinata56:** при переходе на M-OS-1 и росте числа агентов (сейчас 39, будет 45+) единственный файл `MEMORY.md` перестанет справляться. Mem0 self-hosted решает это без отправки данных наружу. Закрывает П-02 Governance-отчёта и Б-06 (agent timeout на длинных задачах).
**Трудозатраты:** 4 часа Analyst'а на RFC. Пилот — после решения по RFC, с выделением backend-dev через Координатора.
