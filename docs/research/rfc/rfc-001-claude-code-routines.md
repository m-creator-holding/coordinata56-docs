# RFC-2026-001: Claude Code Routines для автономных задач coordinata56

**Автор:** ri-analyst
**Дата:** 2026-04-15
**Источник находки:** [findings.md — строка 2026-04-14, Anthropic release](../findings.md)
**Статус:** draft

## 1. Что это
Claude Code Routines — объявлен Anthropic 14.04.2026 в статусе research preview. Позволяет сохранить конфигурацию (промпт + один или несколько репозиториев + MCP-коннекторы + триггеры) и запускать Claude Code автономно в облаке Anthropic без локальной сессии. Три типа триггеров: cron-расписание (минимум 1 час), HTTP API с bearer-токеном, GitHub webhooks (PR/push/issue/release). Работает только при включённом «Claude Code on the web».

## 2. Какую проблему решает
У нас **нет CI**: `.github/workflows/` отсутствует, reviewer и qa запускаются вручную Координатором в сессии. Три конкретные боли:
1. Weekly digest от ri-director (понедельник 10:00) — сейчас держится на памяти Координатора.
2. Нет автоматического запуска reviewer/qa на PR — правило «reviewer до git commit» держится дисциплиной.
3. Ночные долгие задачи (миграции, массовый анализ) требуют живой сессии.

## 3. Что у нас уже есть аналогичного
- **Субагенты** (`~/.claude/agents/*`) — решают декомпозицию, но запускаются только из активной сессии Координатора.
- **Скил `loop`** — рекуррентный запуск промпта в рамках живой сессии. Отключается при закрытии терминала.
- **Скил `schedule`** — создаёт scheduled remote agents (фактически и есть обёртка над Routines/аналогом). Требует проверки — возможно, уже то же самое.
- **CI/CD** — отсутствует полностью.

## 4. Дельта (что нового даёт)
Нужно нам: (а) cron-триггер для weekly digest и ночного qa; (б) GitHub-триггер для автоматического reviewer при открытии PR — закрывает повторяющийся дефект «P0 попадает в main». Не нужно: связка со Slack/Linear/Google Drive. Возможные регрессии: каждый запуск делает fresh clone — теряется локальное состояние БД (Alembic round-trip придётся гонять в контейнере); ветки по умолчанию с префиксом `claude/` — нужно настроить под наш flow.

## 5. Риски
- **Качество**: research preview, не production. API может меняться.
- **Безопасность**: облачный запуск = наш код и `.env` уедут на инфраструктуру Anthropic. Для MVP без PII приемлемо, для боевой фазы с реальными данными Владельца — требует отдельного security-ревью (пересекается с фазой «глубокая безопасность»).
- **Зависимость**: если Anthropic закроет Routines — потеряем автоматизацию, но не код; риск низкий.
- **Лимиты**: 15 запусков/день на аккаунт. Для нас достаточно (reviewer + qa + digest ≈ 5-10 в день).
- **Лицензия**: в рамках подписки Pro/Max/Team/Enterprise; денег сверху не стоит.

## 6. Оценка внедрения
- **Размер пилота**: 1 рабочий день. Настроить 1 routine: «еженедельный digest R&I» с cron `0 10 * * MON`.
- **Кого привлекать**: Координатор (настройка в веб-интерфейсе Claude Code), ri-director (промпт и формат отчёта).
- **Метрика успеха**: digest приходит Владельцу в понедельник 10:00 без участия Координатора 3 недели подряд.
- **Откат**: удалить routine в веб-интерфейсе. Обратимо за 1 минуту, код не затрагивается.

## 7. Рекомендация
**pilot** — низкий риск, нулевая стоимость, решает живую боль (забывающийся weekly digest). Перед пилотом — проверить, не дублирует ли это встроенный скил `schedule`; если дублирует — взять `schedule` как более лёгкий путь.
