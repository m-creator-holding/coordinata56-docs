## Обязательная расшифровка для Владельца (нетехнический уровень)

### Что это делает

Представьте: каждый раз, когда сотрудник приходит на работу, его память стёрта — ему заново рассказывают, кто Вы, что за проект, какие правила. Именно так сейчас работают наши ИИ-субагенты: каждая сессия начинается с нуля, и Координатор тратит время на пересказ контекста.

Mem0 — это общая «картотека» для субагентов. Когда backend-director узнаёт важный факт («в этом проекте запрещён git add -A»), он кладёт карточку в картотеку. В следующей сессии он спрашивает: «что я знаю про git?» — и получает именно эту карточку, а не перечитывает весь архив переписки.

Mem0 устанавливается на наш сервер. Никакие данные никуда не уходят.

### Чем это полезно coordinata56

Конкретная боль сейчас: при запуске M-OS-1 (6+ параллельных Директоров) Координатор будет тратить 20-30% каждого вызова на пересказ фактов, которые субагент «знал» в прошлый раз. С ростом проекта это станет блокером.

Конкретная экономия от Mem0: субагент backend-director в начале сессии делает один запрос к Mem0 («что ты знаешь про этот проект?») и получает сжатый список фактов вместо перечитывания 5-7 файлов MEMORY.md + регламентов.

### Что предлагаем внедрить

Минимальный пилот (2 рабочих дня):
1. Установить Mem0 self-hosted на сервер проекта с векторной БД ChromaDB (встроена, не требует отдельного сервиса).
2. Подключить двух субагентов: backend-director и reviewer.
3. Проверить: субагент при старте сессии получает из Mem0 свой контекст вместо перечитывания файлов.
4. Метрика успеха — ниже в разделе 7.

Если пилот успешен — поэтапно подключаем остальных Директоров.

## 1. Проблема

### 1.1 Источники боли

Задокументировано в Governance-отчёте 2026-04-17:
- **П-02** (проблема процессов): каждая новая сессия субагента начинается без контекста предыдущей. Координатор вынужден пересказывать состояние проекта, решения Владельца, активные ограничения.
- **Б-06** (блокер): при масштабировании до 6+ параллельных Директоров (M-OS-1+) пересказ контекста займёт бо́льшую часть каждого вызова и станет операционным блокером.

### 1.2 Текущее состояние

Память сейчас реализована через файловую систему:
- `/root/.claude/projects/-root/memory/MEMORY.md` — индекс
- `/root/.claude/projects/-root/memory/*.md` — отдельные файлы по темам

Работает для Координатора, но имеет три структурных ограничения:
1. **Линейное чтение**: чтобы «вспомнить» факт, субагент читает весь файл целиком — токены тратятся на нерелевантный контент.
2. **Нет разграничения по агентам**: Директор по backend читает тот же MEMORY.md, что и Координатор; нет namespace'а «только для backend».
3. **Нет поиска по смыслу**: найти «всё, что мы знаем про миграции» — только вручную.

## 2. Решение

### 2.1 Mem0 self-hosted

**Mem0** (github.com/mem0ai/mem0, лицензия Apache-2.0) — open-source библиотека для управления памятью агентов. Устанавливается как Python-библиотека в существующий backend-сервис. Не требует отдельного запущенного сервиса — запускается in-process.

Три слоя хранилища в конфигурации пилота:
- **Векторный** (ChromaDB, встроен): поиск по смыслу — «что мы знаем про миграции?»
- **KV-хранилище**: быстрые факты типа «Владелец = Мартин», «Phase 3 = закрыта»
- **Граф** (опционально, не в пилоте): связи между фактами — не включаем в пилот, лишняя сложность

Mem0 Cloud (SaaS-версия) **исключён полностью** — см. раздел 6.

### 2.2 Жёсткий запрет Mem0 Cloud

Согласно решению Владельца (Telegram msg 1130-1136, 2026-04-17): **подключение Mem0 Cloud запрещено. Использовать исключительно локальную установку (self-hosted).**

Mem0 self-hosted при правильной конфигурации не делает исходящих сетевых запросов к mem0.ai или другим внешним сервисам. Обязательная проверка в пилоте: сетевой дамп (tcpdump или аналог) во время работы — убедиться, что исходящий трафик идёт только к ChromaDB localhost.

## 3. Альтернативы, которые отклонили

### 3.1 Самописная векторная БД (ChromaDB напрямую)

Что это: написать собственный слой поверх ChromaDB без Mem0.

Почему отклонили: Mem0 решает не только хранение, но и «умное» добавление памяти (дедупликация, обновление устаревших фактов, извлечение только релевантного). Самописное решение потребует написания этой логики с нуля — нарушение принципа «сначала простота» (Владелец msg 817). Оценка дополнительных трудозатрат: +3-5 дней против 2 дней с Mem0.

### 3.2 LangGraph Memory

Что это: встроенный механизм памяти фреймворка LangGraph (short-term checkpointer + long-term JSON store).

Почему отклонили: мы не используем LangGraph в стеке — тащить целый фреймворк ради одной функции нарушает принцип минимального скоупа. Плюс LangGraph Memory привязывает нас к их архитектуре thread/namespace, несовместимой с нашей системой субагентов Claude Code.

### 3.3 Anthropic Memory Tool (встроенный)

Что это: нативный инструмент памяти от Anthropic, если появится в API.

Почему отклонили (на момент RFC): на дату 2026-04-17 такого инструмента нет в production API. Ждать — значит блокировать решение боли П-02. Если Anthropic выпустит — отдельный RFC с миграцией.

## 4. Архитектура пилота

### 4.1 Векторная БД

Выбор: **ChromaDB** (встроена в mem0ai пакет, запускается in-process).

Почему не Qdrant:
- Qdrant требует отдельного запущенного сервиса (Docker или бинарник) — лишний операционный компонент для пилота.
- ChromaDB запускается в той же Python-среде, данные хранятся локально в директории `data/mem0_chroma/`.
- Для пилота 2-3 субагентов объём данных ничтожный — ChromaDB более чем достаточна.

При масштабировании свыше 10 субагентов и/или объёме >100k записей — пересмотреть в пользу Qdrant (отдельный RFC).

### 4.2 Схема памяти: shared vs per-subagent vs hybrid

Выбор: **hybrid** — два слоя.

**Слой 1 — project-wide (общий):**
- Namespace: `coordinata56_project`
- Содержимое: факты проекта — фазы, правила из CLAUDE.md, решения Владельца, ADR-итоги
- Кто пишет: Координатор (после каждого значимого решения)
- Кто читает: любой субагент при инициализации сессии

**Слой 2 — per-subagent (персональный):**
- Namespace: `agent_{role}` (например `agent_backend_director`, `agent_reviewer`)
- Содержимое: специфичные факты роли — «в этой сессии был баг в миграции X», «паттерн Y работает в нашем стеке»
- Кто пишет: сам субагент через Координатора-транспорт
- Кто читает: только этот субагент

Почему не только shared: Директору по backend не нужно знать детали UX-решений. Namespace изолирует шум.

Почему не только per-subagent: общие факты проекта дублировать в 17 namespace'ах — избыточно и рассинхронизируется.

### 4.3 Как Координатор записывает и читает

**Запись (при завершении сессии или по событию):**
```
POST /memory/add
{
  "agent_id": "backend_director",  # или "project" для общего слоя
  "user_id": "coordinata56",
  "messages": [{"role": "user", "content": "<факт для запоминания>"}]
}
```

**Чтение (при инициализации сессии субагента):**
```
GET /memory/search
{
  "agent_id": "backend_director",
  "user_id": "coordinata56",
  "query": "текущий контекст проекта и мои задачи"
}
```

В пилоте взаимодействие реализуется как Python-функции, вызываемые из системного промпта субагента через MCP-инструмент или напрямую из Координатора перед передачей контекста Директору.

**Триггеры записи:**
- После каждого завершённого RFC или пилота: Координатор сохраняет итог в project-wide слой.
- После каждой значимой инструкции Владельца: Координатор сохраняет в project-wide.
- Субагент сам инициирует запись через Координатора (паттерн «Координатор-транспорт», регламент v1.6) — не напрямую.

### 4.4 Где хранится

Хранение: **только local**, папка `/root/coordinata56/data/mem0_chroma/` на нашем сервере.

Обоснование: ст. 45а CODE_OF_LAWS — запрет живых внешних интеграций. Mem0 self-hosted при конфигурации с локальной ChromaDB не делает сетевых запросов. Проверяется в пилоте (п. 7.2).

## 5. Первые субагенты пилота

Предложение — 2 субагента для первой волны:

**1. backend-director** (приоритет 1)
Обоснование: наибольшая контекстная нагрузка — знает правила по коду, миграциям, API-форматам (ADR 0004-0007), текущую фазу. Сейчас перечитывает CLAUDE.md + CODE_OF_LAWS + ADR при каждой сессии. Выигрыш очевиден и измерим.

**2. reviewer** (приоритет 2)
Обоснование: reviewer работает с одним и тем же чеклистом правил (P0/P1 паттерны из CLAUDE.md). Если Mem0 даст ему «помнить» типичные ошибки конкретного проекта — качество ревью вырастет без роста промпта.

Третий кандидат (резерв, не в пилоте): **governance-auditor** — контекстно-зависим, но его сессии редки. Добавить в волну 2 после оценки результатов волны 1.

## 7. Метрики пилота (определение успеха)

### 7.1 Основная метрика

**Сокращение токенов на контекст при инициализации субагента:**
- Базовая линия (до пилота): замерить размер системного промпта + прочитанных файлов при старте backend-director.
- Цель: сокращение на 30%+ при сохранении качества работы.

### 7.2 Обязательная проверка: no outbound traffic

Во время пилота: `tcpdump -i any host mem0.ai or host api.mem0.ai` должен показать 0 пакетов. Если пакеты есть — пилот немедленно останавливается, находка переходит в Rejected.

### 7.3 Дополнительные метрики

| Метрика | Цель |
|---|---|
| Релевантность извлечённых воспоминаний | >= 80% извлечённых фактов реально используется в ответе субагента |
| Скорость поиска | <= 500 мс на запрос (ChromaDB in-process) |
| Отсутствие ошибок интеграции | 0 сбоев при инициализации субагента в 10+ тестовых запусках |
| Обратная совместимость | старый MEMORY.md продолжает работать параллельно (не удаляем до adopt) |

## 8. Риски и план отката

### 8.1 Риски

| Риск | Вероятность | Последствие | Митигация |
|---|---|---|---|
| Mem0 делает скрытые исходящие запросы | Низкая | Критично — нарушение ст. 45а | Обязательный tcpdump в п. 7.2 |
| API mem0ai меняется (версия 0.x) | Средняя | Пилот сломан | Фиксируем версию в requirements.txt, автообновление запрещено |
| ChromaDB медленнее ожиданий | Низкая | Задержки инициализации | Порог 500 мс в метрике; если выше — замена на Qdrant |
| Субагенты «доверяют» устаревшей памяти | Средняя | Неверные решения на старых данных | TTL на записи (30 дней), ручная инвалидация через Координатора |
| Пилот провалился (ни одна метрика не выполнена) | Низкая | 2 дня работы потеряны | Откат: убрать Mem0 из промптов, файловая схема остаётся нетронутой |

### 8.2 План отката

Откат занимает < 1 часа:
1. Убрать вызовы Mem0 из системных промптов двух пилотных субагентов.
2. Удалить папку `data/mem0_chroma/` (только пилотные данные, MEMORY.md не затронут).
3. Удалить зависимости mem0ai и chromadb из requirements.txt.
4. Зафиксировать статус в findings.md: Rejected + причина.

Старый файловый механизм (`MEMORY.md` + `memory/*.md`) работает параллельно на протяжении всего пилота и не удаляется до явного решения adopt.

## 9. Внешние зависимости (Python-библиотеки)

| Библиотека | Версия (пилот) | Лицензия | Размер | Известные CVE |
|---|---|---|---|---|
| `mem0ai` | `0.1.x` (фиксируем точную при установке) | Apache-2.0 | ~15 MB с зависимостями | Нет публичных CVE на 2026-04-17; проверить перед установкой через `pip-audit` |
| `chromadb` | `0.5.x` (вытягивается как зависимость mem0ai) | Apache-2.0 | ~40 MB | Нет критичных CVE; разработка активна |
| `openai` | уже установлен в проекте или `1.x` | MIT | ~5 MB | Нет критичных CVE |

Примечания:
- `openai` тащится mem0ai для embedding-функций. В нашей конфигурации embedding вычисляется локально (модель `all-MiniLM-L6-v2` через sentence-transformers) либо через уже имеющийся Anthropic-клиент — зависит от финального выбора в пилоте. Уточняется на этапе установки.
- `sentence-transformers` (если выбран локальный embedding): Apache-2.0, ~500 MB с моделью. Это самая тяжёлая зависимость пилота — при нежелательности скачивания модели использовать embedding через Anthropic API (нет дополнительных зависимостей, но есть токен-стоимость).
- Все лицензии (Apache-2.0, MIT) совместимы с нашим некоммерческим внутренним использованием.
- Перед установкой: запустить `pip-audit` на предмет CVE в зафиксированных версиях.

### A1. Запрет Anthropic API для embedding

**Решение:** embedding выполняется **исключительно** через локальную библиотеку `sentence-transformers`. Anthropic API (`/v1/embeddings`) — **запрещён**.

**Обоснование:**
- Принцип «данные проекта не уходят во внешние сервисы» — та же причина, по которой Mem0 Cloud отвергнут (§2.2 оригинального RFC).
- Использование Anthropic API для embedding передаёт текст воспоминаний (факты проекта, решения Владельца, контекст субагентов) на серверы Anthropic — нарушение ст. 45а CODE_OF_LAWS v2.1 в части «живых сетевых вызовов к сторонним системам».
- Аудитор (msg 1152) корректно указал: tcpdump-критерий §7.2 проверял только `mem0.ai / api.mem0.ai`, оставляя канал `api.anthropic.com` вне проверки — это сетевой обход.

**Нормативное основание запрета:**
> Ст. 45а CODE_OF_LAWS v2.1: живые внешние интеграции — реальные сетевые вызовы к сторонним системам — запрещены до прохождения production-gate проекта.
> Решение Владельца Telegram msg 1147, 2026-04-17: «embedding только локальный sentence-transformers».

### A2. Embedding-модель для русского текста

**Выбранная модель:** `paraphrase-multilingual-mpnet-base-v2` (HuggingFace, sentence-transformers)

**Характеристики:**

| Параметр | Значение |
|---|---|
| Размер модели на диске | ~1.1 GB (fp32) |
| Размерность вектора | 768 |
| Поддержка языков | 50+ языков, включая русский |
| Качество на русском | Высокое — обучена специально на multilingual-STS, одна из лучших для русских технических текстов |
| Скорость inference | ~10-30 мс/запрос на CPU (для коротких текстов памяти) |
| Лицензия | Apache-2.0 |

**Альтернатива:** `paraphrase-multilingual-MiniLM-L12-v2` — легче (~470 MB), быстрее, качество чуть ниже. Рекомендуется если 1.1 GB неприемлем на сервере.

**Важно:** модель скачивается **один раз при установке** (`pip install sentence-transformers` + первый вызов `SentenceTransformer('...')`). В runtime никаких исходящих запросов к HuggingFace Hub нет — модель работает полностью локально из кэша.

**Конфигурация Mem0 с локальным embedding:**
```python
from mem0 import Memory

config = {
    "embedder": {
        "provider": "huggingface",
        "config": {
            "model": "sentence-transformers/paraphrase-multilingual-mpnet-base-v2"
        }
    },
    "vector_store": {
        "provider": "chroma",
        "config": {
            "collection_name": "coordinata56",
            "path": "/root/coordinata56/data/mem0_chroma"
        }
    }
}

memory = Memory.from_config(config)
```

### A3. Расширенный tcpdump-критерий (замена §7.2)

**Предыдущая версия §7.2 (недостаточна):**
```
tcpdump -i any host mem0.ai or host api.mem0.ai
```

**Новая версия §7.2 (обязательна для пилота):**

```bash
# Полный egress-мониторинг в runtime — запускать ПАРАЛЛЕЛЬНО с пилотом
tcpdump -i any -n \
  '(host mem0.ai or host api.mem0.ai) or
   (host api.anthropic.com) or
   (host api.openai.com) or
   (host api.huggingface.co)'
```

Результат: **0 пакетов** по каждому из хостов во время работы Mem0 в runtime.

**Разрешённый egress-whitelist для пилота (всё прочее — запрещено):**

| Хост | Протокол | Назначение | Разрешён? |
|---|---|---|---|
| `localhost` / `127.0.0.1` | любой | ChromaDB in-process | Да |
| `localhost` / `127.0.0.1` | любой | FastAPI backend | Да |
| Внутренняя сеть сервера | любой | Внутренние ресурсы проекта | Да |
| `api.anthropic.com` | HTTPS | LLM inference (не embedding) | **Только для LLM-вызовов субагентов; embedding — запрещён** |
| `api.mem0.ai`, `mem0.ai` | любой | Mem0 Cloud | Запрещён |
| `api.openai.com` | любой | OpenAI API | Запрещён |
| `api.huggingface.co` | любой | HuggingFace runtime | Запрещён в runtime; разрешён только при первоначальной установке |

**Примечание по `api.anthropic.com`:** LLM-вызовы субагентов через этот хост — рабочий режим Claude Code, не блокируются. Блокируется **исключительно** использование `/v1/embeddings` endpoint через embedding-конфиг Mem0. Проверка: убедиться что конфиг Mem0 не содержит `provider: "anthropic"` в секции `embedder`.

### A5. Дополнительный шаг пилота: проверка no-outbound sentence-transformers

В план развёртывания (§10) добавить **Шаг 3а** между Шагами 3 и 4:

| Шаг | Что делается | Исполнитель | Критерий готовности |
|---|---|---|---|
| 3а | Убедиться что sentence-transformers не делает исходящих запросов в runtime: запустить `strace -e trace=network` или tcpdump во время 10 embedding-запросов к уже установленной модели | qa | 0 исходящих соединений к внешним хостам; все сетевые операции — только loopback |

Если обнаружен outbound к `api.huggingface.co` в runtime — это **инцидент по ст. 45б CODE_OF_LAWS v2.1** (доклад Владельцу через Координатора). Возможная причина: sentence-transformers пытается проверить обновление модели. Митигация: `TRANSFORMERS_OFFLINE=1` в env.
