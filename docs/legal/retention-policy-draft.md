## Контекст

152-ФЗ ст. 5 ч. 4: «Хранение персональных данных должно осуществляться в форме, позволяющей определить субъекта персональных данных, не дольше, чем этого требуют цели обработки персональных данных».

152-ФЗ ст. 21 ч. 1: При достижении цели обработки оператор обязан прекратить обработку и уничтожить ПДн в срок, не превышающий 30 дней (если иное не предусмотрено договором, законом, лицензионным соглашением).

## Категории retention по сроку

**Немедленно при деактивации:**
- password_hash

**1 год:**
- Notification.title / body / payload (если содержат ПДн)
- SubagentEvent.payload (если содержит ПДн)

**3 года (исковая давность ГК):**
- AuditLog.changes_json

**5 лет (налоговый архив НК РФ + коттеджное строительство 214-ФЗ):**
- User (при статусе «клиент»)
- Contractor.contacts_json
- Payment
- BPMWorkflowInstance

**75 лет (ТК РФ + 125-ФЗ архивное дело — TBD юрист):**
- User (при статусе «сотрудник») — если M-OS будет обрабатывать ПДн штатных сотрудников

## Механизм реализации (рекомендации backend-dev)

### Техническое решение

```sql
-- Добавить в модели поля для автоматизации retention:
-- User
ALTER TABLE users ADD COLUMN deactivated_at TIMESTAMPTZ;
ALTER TABLE users ADD COLUMN delete_after TIMESTAMPTZ
    GENERATED ALWAYS AS (deactivated_at + INTERVAL '5 years') STORED;

-- Contractor
ALTER TABLE contractors ADD COLUMN closed_at TIMESTAMPTZ;
ALTER TABLE contractors ADD COLUMN contacts_delete_after TIMESTAMPTZ
    GENERATED ALWAYS AS (closed_at + INTERVAL '5 years') STORED;

-- Notification
ALTER TABLE notifications ADD COLUMN delete_after TIMESTAMPTZ
    GENERATED ALWAYS AS (
        CASE WHEN status = 'archived' 
        THEN updated_at + INTERVAL '1 year'
        ELSE NULL END
    ) STORED;
```

### Scheduled job (Celery / pg_cron)

```
# Ежемесячный retention job:
# 1. SELECT * FROM users WHERE delete_after < NOW()
#    → обезличить или удалить по правилу категории
# 2. SELECT * FROM contractors WHERE contacts_delete_after < NOW()
#    → contacts_json = NULL или anonymized
# 3. SELECT * FROM notifications WHERE delete_after < NOW()
#    → DELETE (уведомления не нужны после retention)
# 4. SELECT * FROM audit_log WHERE created_at < NOW() - INTERVAL '3 years'
#    → DELETE или архив в cold storage
```

## Открытые вопросы для юриста

1. **User = сотрудник vs клиент:** Если M-OS хранит данные штатных работников холдинга, применяется ТК РФ ст. 87 и 125-ФЗ о личных делах (75 лет). Требуется чёткая категоризация пользователей в системе.

2. **Подрядчик-ИП:** ИНН ИП и его контактные данные — ПДн физического лица. Нужна отдельная форма согласия или иное основание ст. 6 ч. 1 152-ФЗ.

3. **214-ФЗ и хранение по МКД:** Если M-OS используется для МКД — ст. 17 214-ФЗ устанавливает обязанность хранить документацию в течение 5 лет после передачи объекта. Может влиять на retention платёжных и договорных записей.

4. **AuditLog как юридически значимый архив:** Если audit_log используется как доказательство в споре, его retention не может быть менее срока исковой давности. Нужна квалификация юриста.

5. **Уведомление РКН:** При обработке ПДн сотрудников — уведомление Роскомнадзора обязательно (152-ФЗ ст. 22). При обработке только контрагентов — возможны исключения ст. 22 ч. 2. Требует оценки по реальному составу обработки.
