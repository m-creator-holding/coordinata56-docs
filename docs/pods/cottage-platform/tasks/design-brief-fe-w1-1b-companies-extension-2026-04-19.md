## 0. Назначение

Дорисовать недостающие wireframes для расширения `/admin/companies` согласно
Решению Владельца Q7 (7 обязательных полей) и паттерну Developer Mode
(версионирование + правая панель истории + readonly-mode для бухгалтера).

Базовая страница уже существует (FE-W1-1). Нужны **новые экраны-приращения**,
а не переработка с нуля.

---

## 1. Бизнес-цель

Владелец и бухгалтер должны различать 5 юрлиц холдинга (карьер, металл,
8 АЗС, МКД-девелопер, коттеджный девелопер) по полной 7-польной
конфигурации; Владелец меняет любое поле и сразу видит новую версию v(N+1)
в правой панели; бухгалтер открывает ту же карточку в режиме readonly.

---

## 2. Что дорисовать (3 экрана-приращения)

### 2.1 List View — добавить колонки сегмента / НДС-режима / версии

Существующая таблица `CompaniesListPage` показывает: название / тип / ИНН /
is_active. Нужно добавить колонки:

- **Сегмент** (business_segment — из enum: Коттеджный / АЗС / Карьер /
  Металлообработка / МКД-девелопер) — Badge variant=secondary
- **НДС-режим** (vat_regime — ОСН / УСН 6% / УСН 15% / Патент) — Badge
- **Версия** (v1, v2, v3 …) — Badge variant=outline
- **Статус** (Активна / Архивная) — Badge variant=success / muted

Старые колонки (ИНН/тип) оставить. Фильтр «Все сегменты» — добавить Select
в toolbar.

**Для бухгалтера:** row-level isolation — видит только своё юрлицо
(одна строка). Кнопка «+ Добавить юрлицо» — скрыта.

### 2.2 Detail View — 3 вкладки + правая панель истории версий

Сейчас страница показывает одну плоскую форму. Нужно разбить на Tabs:

1. **Основные данные** — title (full_name), business_segment (Select),
   company_director_id (Combobox из users)
2. **Финансы** — vat_regime (RadioGroup: ОСН / УСН 6% / УСН 15% / Патент),
   bank_primary_account (Input 20 цифр), default_currency (Select: RUB),
   payment_overrun_limit_pct (Input number 1–100 с суффиксом %),
   approval_amount_threshold (Input number ≥ 0 с суффиксом ₽)
3. **Аудит** — лог изменений (sprint 1.1B можно свести к простому списку)

**Правая панель «История версий»** — фиксированная ширина ~280 px:
- Таймлайн v3 / v2 / v1 с датой, автором и коротким комментарием
  («Изменён НДС-режим», «Создана запись»)
- Активная версия помечена точкой и текстом «Текущая»
- У каждой неактивной версии — кнопка «Откатить»

### 2.3 Readonly-mode для бухгалтера (accountant)

- Banner поверх формы: «У вас режим просмотра. Для изменений — обратитесь
  к директору.» (variant=info, не destructive)
- Все поля — disabled + `bg-muted`
- Кнопки «Сохранить изменения» / «Отмена» — **скрыты полностью** (не
  disabled, а `display: none`)
- Правая панель истории — видна (только просмотр), кнопка «Откатить»
  скрыта для каждой версии
- Вкладки Основные / Финансы / Аудит — все видны, все readonly

## 5. Acceptance Criteria (≥3 тестовых сценария)

AC-1. **List View — бухгалтер видит только своё юрлицо.**
Given MSW-user с ролью accountant, привязанный к ООО «Координата 56».
When открывает `/admin/companies`.
Then таблица содержит ровно 1 строку; кнопка «+ Добавить юрлицо» отсутствует;
колонки Сегмент/НДС-режим/Версия/Статус присутствуют.

AC-2. **Detail View — Владелец сохраняет новую версию.**
Given Владелец на `/admin/companies/:id` активная вкладка Финансы v2.
When меняет vat_regime с ОСН на УСН 15% и кликает «Сохранить изменения».
Then Toast «Юрлицо обновлено. Версия v3 активна.»; правая панель истории
показывает v3 сверху с комментарием/автором/датой; v2 получает кнопку
«Откатить».

AC-3. **Detail View — бухгалтер readonly.**
Given MSW-user accountant на `/admin/companies/:id`.
Then banner «У вас режим просмотра. Для изменений — обратитесь к директору.»
виден; все Input/Select/Radio — disabled; кнопки «Сохранить изменения» и
«Отмена» отсутствуют в DOM; правая панель истории видна без кнопок
«Откатить».

AC-4. **Откат к предыдущей версии.**
Given Владелец на `/admin/companies/:id` v3, в правой панели v2 с кнопкой
«Откатить».
When кликает «Откатить» на v2.
Then AlertDialog «Откатить к версии v2 (10.04.2026, Мартин)?»; после
подтверждения — создаётся v4 с данными v2; Toast «Откат к версии v2
выполнен. Текущая версия: v4.».

## 7. Открытые вопросы к design-director

DQ-1. **Layout правой панели на узких экранах** (< 1280 px) — collapsible
в Sheet справа или всегда видимый? Рекомендация Директора: collapsible
Sheet, кнопка «История версий» в toolbar формы.

DQ-2. **Вкладка Аудит** — в sprint 1.1B показывать плоский список событий
(создал/изменил/откатил) или полноценный log с фильтрами? Рекомендация:
плоский список из `audit_log` API, фильтры — M-OS-1.1C.

DQ-3. **Иконки для сегментов** — рисуем уникальную иконку каждому
(карьер / АЗС / металл / МКД / коттедж) или обходимся без? Рекомендация:
без иконок в sprint 1.1B, Badge достаточно; иконки — отдельная задача
design-system в M-OS-2.
