# Wireframes: Houses — операционные экраны стройки

- **Версия:** 1.0
- **Дата:** 2026-04-18
- **Автор:** designer (L4) / приёмка: design-director
- **Батч-ID:** M-OS-1.x-design-houses
- **Под-фаза:** M-OS-1 (pod: cottage-platform)
- **Статус:** готов к handoff → frontend-director
- **Персоны:** Прораб (primary), Менеджер стройки (secondary), Владелец (read-only)

---

## Навигационная карта

```
/houses                          → Экран 1: Houses List
/houses/new                      → Экран 3: House Create Form
/houses/:id                      → Экран 2: House Detail (вкладка Общее)
/houses/:id?tab=stages           → Экран 2: House Detail (вкладка Стадии)
/houses/:id?tab=budget           → Экран 2: House Detail (вкладка Бюджет)
/houses/:id?tab=contracts        → Экран 2: House Detail (вкладка Контракты)
/houses/:id?tab=journal          → Экран 2: House Detail (вкладка Журнал)
/houses (selection mode)         → Экран 4: Batch Update (активируется поверх списка)
```

Экран 4 (Batch Update) — не отдельный роут, а режим поверх экрана 1: при выборе ≥2 домов появляется Bottom-action-bar с операциями.

---

## Принятые решения

| # | Решение | Выбрано | Обоснование |
|---|---|---|---|
| H1 | Список домов: таблица или сетка | Toggle Table/Grid — дефолт таблица | Прораб работает с 85 домами и сравнивает статусы: таблица даёт сортировку и плотность. Grid — для визуального обзора по типам. Toggle сохраняет предпочтение в localStorage. |
| H2 | Прогресс-бар в списке | Тонкая полоска (Progress bar из shadcn/ui) в колонке, без цифр | Цифра «%» в числе занимает место; полоска читается моментально; числовое значение — в тултипе и на детальной карточке. |
| H3 | Stage badges | 8 цветных Badge с иконкой, по одному на строку | Цвет + иконка + текст — принцип DS v1.0 «не только цвет». Цветовая шкала от нейтрального (идея) к зелёному (сдан). |
| H4 | Batch Update | Bottom-action-bar фиксированный при прокрутке | Не Dialog и не отдельная страница: прораб видит список и действие одновременно. Паттерн из Gmail / Jira. |
| H5 | House Detail — вкладки | Tabs (shadcn/ui) — 5 вкладок | Аналогично Admin-паттерну (карточка компании с вкладками, D10). Единообразие интерфейса. |
| H6 | Переход между стадиями | Кнопка «Перевести в стадию: [следующая]» + confirmation Dialog | PATCH /houses/{id} с аудитом требует явного намерения. Авто-переход запрещён. |
| H7 | Фото | Placeholder-секция с Info-баннером | Мобильное приложение — отдельная фаза. Кнопка «Прикрепить фото» — disabled с баннером «Доступно в мобильном приложении». |
| H8 | Экспорт списка | Кнопка «Экспорт» → выпадающий список (CSV / Excel) | Паттерн прост, не требует нового компонента. Excel — формат прорабов. |
| H9 | Создание дома — контракты | Секция «Контракты» на форме создания — опциональная, свёрнута | Контракты появятся позже; форма не блокирует создание дома без них. Accordion скрывает секцию по умолчанию. |
| H10 | Ответственный (прораб) в списке | Avatar + ФИО в колонке | Визуальная идентификация быстрее, чем только текст; Avatar из shadcn/ui. |

## Палитра стадий (Stage Badge Palette)

Используется во всех 4 экранах. Базируется на DS v1.0 semantic-цветах.

| Стадия | Badge-цвет (semantic) | Иконка (Lucide) | Логика |
|---|---|---|---|
| Идея | `--color-text-disabled` / серый | `Lightbulb` | Ещё не начали |
| Земля | `--color-warning-bg` / жёлтый-светлый | `Shovel` (или `MapPin`) | Подготовка участка |
| Фундамент | `--color-warning` / жёлтый | `Layers` | Активная стройка, ранняя стадия |
| Стены | `--color-warning` / жёлтый-тёплый | `Building2` | Активная стройка |
| Кровля | `--color-info-bg` / голубой-светлый | `Home` | Коробка готова |
| Инженерия | `--color-info` / синий | `Zap` | Внутренние работы |
| Финиш | `--color-success-bg` / зелёный-светлый | `Paintbrush` | Финальные отделочные работы |
| Сдан | `--color-success` / зелёный | `CheckCircle2` | Объект сдан |

## Экран 1. Houses List — `/houses`

### Persona и сценарий

**Персона:** Прораб / Менеджер стройки.
**Сценарий:** Прораб открывает список домов утром, фильтрует по своим объектам (`responsible_user = я`), сортирует по стадии, визуально оценивает прогресс. Нажимает на дом — переходит в карточку. Выбирает несколько домов для массового обновления стадии.

### Состояния экрана (5 состояний UI)

| Состояние | Описание |
|---|---|
| Loading | MOSTable со skeleton-строками (shimmer, 5 строк), фильтры disabled |
| Empty (нет домов) | Иконка `Building2` (lg) + «Дома ещё не добавлены» + кнопка «+ Добавить дом» |
| Empty (фильтр дал 0) | Иконка `SearchX` + «Ничего не найдено по фильтрам» + кнопка «Сбросить фильтры» |
| Populated | Штатный режим: таблица или сетка с данными |
| Error | Banner над таблицей: `--color-danger-bg`, иконка `XCircle`, текст «Не удалось загрузить список домов», кнопка «Повторить» |

## Экран 2. House Detail — `/houses/:id`

### Persona и сценарий

**Персона:** Прораб.
**Сценарий:** Прораб открывает карточку конкретного дома — видит текущую стадию, % готовности, контакты ответственного. Переходит на вкладку «Стадии» — проверяет план vs факт. Переходит на «Бюджет» — смотрит отклонение от плана. Переводит дом в следующую стадию кнопкой «Перевести в [следующую стадию]».

## Открытые вопросы

### К backend

| # | Вопрос | Критичность |
|---|---|---|
| B1 | `PATCH /api/v1/houses/batch-update` — возвращает ли partial failure (`updated`/`failed`) или транзакция all-or-nothing? Для UX «partial error» нужен первый вариант. | P1 |
| B2 | Есть ли отдельный эндпоинт для аудит-лога дома (`/houses/:id/audit-log`) или берём из общего аудит-лога с фильтром по `resource_id`? | P2 |
| B3 | Поле `progress_pct` в модели House вычисляется на backend (по стадиям) или вводится вручную прорабом? Влияет на wireframe вкладки «Стадии». | P1 |
| B4 | Экспорт (`/houses/export`) — планируется в M-OS-1 или позже? Если позже — кнопка «Экспорт» в wireframe делается disabled с баннером. | P2 |
| B5 | `house_type` хранится как enum (A/B/C/D) или как FK на таблицу типов? Влияет на фильтр в Houses List. | P2 |
