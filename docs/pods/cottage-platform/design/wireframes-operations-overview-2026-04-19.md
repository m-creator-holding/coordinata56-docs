## Принятые решения

| # | Точка выбора | Выбрано | Обоснование |
|---|---|---|---|
| O1 | Сетка домов: SVG-матрица vs CSS-grid квадратов | CSS-grid 10×9 квадратов 28px | shadcn/ui не содержит SVG-компонента; CSS-grid — реализуется через стандартные div + Badge-цвета из палитры |
| O2 | KPI: горизонтальный ряд vs 2×2 | Desktop: 4 в ряд. Mobile: стек 2×2 | На десктопе плотность выше; на мобиле 4 в ряд — слишком узкие карточки |
| O3 | Аудит-лента: фиксированная высота vs расширяемая | ScrollArea фиксированная высота 280px | Дашборд не должен бесконечно расти; ScrollArea даёт прокрутку без потери общей компоновки |
| O4 | Финансовый snapshot: position | После KPI, перед сеткой домов (desktop) | owner/director сначала читает финансы, потом детализирует по домам |
| O5 | Блокирующие вопросы: отдельная секция vs Badge в сетке | Отдельная секция внизу + Badge «!» на квадрате в сетке | Двойное сигнализирование: сетка даёт геолокацию проблемы, секция — список для действий |
| O6 | Mobile sidebar | Bottom tab bar 4 иконки (Дашборд / Дома / Задачи / Профиль) | Sidebar на планшете foreman неудобен; bottom tabs — стандарт мобильного рабочего UI |
| O7 | Сетка домов mobile | 5 колонок, квадраты 40px (крупнее для тапа пальцем) | Пальцем на 28px сложно; 40px — минимальный touch target по HIG |

## Навигационная карта

```
/projects/{projectId}/overview          ← этот файл
  ├── клик на квадрат сетки             → /houses/{houseId}
  ├── «Открыть» в блокирующих вопросах  → /houses/{houseId}
  └── sidebar / bottom-tabs             → /projects/{projectId}/houses
                                        → /reports/financial (owner/director/accountant)
```

## Экран 1. Project Overview Dashboard — `/projects/{projectId}/overview`

### Persona и сценарий

**Персона A — Владелец/Директор (owner/director):**
Утром открывает дашборд, видит 4 KPI сверху, читает финансовый snapshot (план/факт/дельта), бросает взгляд на сетку домов (цветовая картина строительства), проверяет блокирующие вопросы и аудит-ленту. Принимает решение о звонке прорабу.

**Персона B — Бухгалтер (accountant):**
Открывает дашборд. Видит KPI и финансовый snapshot (основной интерес), сетку домов — в режиме просмотра. Баннер «Режим просмотра» на верхней части страницы.

**Персона C — Прораб (foreman):**
Открывает с планшета на объекте. Видит KPI (Домов / Сдано / В работе / Заблокировано), сетку домов (тапает по своим объектам), блокирующие вопросы. Финансовый snapshot — полностью скрыт (не баннер, просто отсутствует на экране).

### States

**Loading state:**
```
KPI-карточки: Skeleton (shadcn/ui) 4 блока
Сетка домов:  Skeleton прямоугольник
Финснапшот:   Skeleton 3 строки
Аудит-лента:  Skeleton 5 строк с Avatar
```

**Empty state (нет домов в проекте):**
```
┌─────────────────────────────────────────┐
│  Дома ещё не добавлены в проект         │
│  Обратитесь к администратору или        │
│  импортируйте реестр домов              │
│  [Перейти в настройки проекта]          │
└─────────────────────────────────────────┘
```

**Error state (сервер недоступен):**
```
[Badge: Ошибка загрузки]
Не удалось получить данные. Проверьте соединение.
[Обновить страницу]
```
