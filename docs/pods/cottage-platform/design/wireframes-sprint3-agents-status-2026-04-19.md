## Принятые решения

| # | Точка выбора | Выбрано | Обоснование |
|---|---|---|---|
| A1 | Обновление данных: polling vs SSE | Polling каждые 30 сек + кнопка «Обновить» | SSE требует отдельной инфраструктуры (M-OS-2+); polling достаточно для операционного мониторинга без real-time требований |
| A2 | Детальная карточка: Sheet справа vs отдельный роут | Sheet справа | Пользователь не теряет контекст списка; отдельный роут нужен если нужен deep-link (пока нет требования) |
| A3 | Группировка агентов: дерево vs Badge-фильтр | Badge-фильтр (Группа: Все / Core / Departments / Archive) | 40+ агентов — дерево создаёт визуальный шум; фильтр проще и быстрее для сканирования |
| A4 | Деструктивные действия (деактивация агента) | Не проектируются | Управление статусами субагентов — операция системного уровня (Координатор), не через Admin UI Владельца. Только просмотр. |
| A5 | Анимация статуса Working | Pulsating dot (CSS animation, описание для frontend) | Badge со статическим цветом не передаёт динамику; CSS pulse — минимальная реализация без JS-зависимостей |

## Role Matrix

| Роль | Доступ к /admin/agents | Ограничения |
|---|---|---|
| owner | Полный просмотр всех агентов, детальных карточек | Нет деструктивных действий — только наблюдение |
| director | 403 Forbidden | Операционная карта — только Владелец |
| accountant | 403 Forbidden | Финансовый пользователь; агенты вне их контекста |
| foreman | 403 Forbidden | Прораб не работает с системными операциями |

**Примечание:** /admin/agents доступен только при `role.code === 'owner'`. Любая другая роль → 403 без раскрытия содержимого страницы.

## Экран 1. `/admin/agents` — Операционная карта субагентов

### Persona и сценарий

**Персона A — Владелец (owner):**
Мартин открывает /admin/agents утром для операционного обзора. Хочет понять: сколько агентов активно прямо сейчас, нет ли ошибок. Видит 4 счётчика вверху (Active/Working/Dormant/Error). Замечает, что в Error — 1 агент. Фильтрует по статусу «Error», кликает на строку — открывается Sheet с деталями. Читает последнюю ошибку, принимает решение о действии.

**Статус-индикаторы (цветовые Badge):**
- `[Active] ●` — Badge variant=success, зелёный dot
- `[Working] ◉` — Badge variant=default (синий) + CSS pulsating dot (animation: pulse 2s infinite)
- `[Dormant] ○` — Badge variant=secondary, серый dot
- `[Error] ✕` — Badge variant=destructive, красный

**Кликабельность счётчиков:** нажатие на карточку [Error: 1] применяет фильтр Статус=Error в таблице ниже. Нажатие повторно — сбрасывает фильтр.

### Поля формы / колонки таблицы

| Колонка | Тип UI | Ширина | Примечание |
|---|---|---|---|
| Имя агента | текст + клик на строку | 25% | Клик открывает Sheet (Экран 2) |
| Уровень | текст (L1/L2/L3/L4) | 8% | Иерархический уровень |
| Группа | Badge | 15% | Core / Departments / Archive |
| Статус | Badge с dot | 15% | Цвет по статусу, см. выше |
| Текущая задача | текст или «—» | 30% | Короткий идентификатор задачи или «—» |
| Последнее действие | timestamp | — | Формат: «5 мин назад» / дата если > суток |

**Компоненты shadcn/ui:** `Table`, `TableHeader`, `TableRow`, `TableCell`, `Badge`, `Select` (фильтры Группа, Статус), `Button` (Обновить), `Pagination`, `Card` (счётчики вверху).

**Счётчики-карточки:** 4 штуки в ряд, каждая — `Card` с `CardContent`: большое число + подпись статуса + цветная точка. Клик на карточку = фильтр.

### Состояния UI

| Состояние | Что показывается |
|---|---|
| Loading | Skeleton 8 строк таблицы + skeleton 4 счётчиков |
| Error загрузки | Alert variant=destructive: «Не удалось загрузить данные субагентов. Попробуйте обновить.» + кнопка «Обновить» |
| Фильтр без результата | «Агенты по выбранным фильтрам не найдены.» + ссылка «Сбросить фильтры» |
| Все агенты dormant | Редкий сценарий. Все счётчики кроме Dormant = 0. Нет алертов — это нормальный режим покоя. |
| Обновление данных | Кнопка «Обновить» показывает spinner пока идёт запрос. После — «Обновлено только что». |

## Экран 2. Sheet — Детальная карточка субагента

### Поля Sheet

| Блок | Содержимое | Тип UI |
|---|---|---|
| Заголовок | Имя агента + Badge Статус | SheetHeader + Badge |
| Метаданные | Уровень, Группа | текст с разделителем ◆ |
| Описание роли | Текст описания из конфигурации агента | параграф |
| Активная задача | Краткое описание или «нет активной задачи» | текст (muted если нет задачи) |
| Запущен | Datetime start | текст |
| Последнее действие | Relative time («5 мин назад») | текст |
| История активности | ScrollArea: список событий (timestamp, level, описание) | ScrollArea + таблица или stack |
| Кнопка Закрыть | Закрывает Sheet | Button variant=outline |

**Компоненты shadcn/ui:** `Sheet`, `SheetHeader`, `SheetTitle`, `SheetContent`, `SheetClose`, `Badge`, `ScrollArea`, `Separator`, `Button`.

**Нет деструктивных кнопок:** Sheet только для просмотра. Нет «Деактивировать», «Перезапустить» — управление агентами вне Admin UI.

## Acceptance Criteria

| # | Критерий | Приоритет |
|---|---|---|
| AC-AGT1 | GET /api/v1/agents/summary возвращает 4 числа (active, working, dormant, error). Карточки-счётчики обновляются одновременно с таблицей. | P0 |
| AC-AGT2 | Клик на карточку-счётчик применяет фильтр Статус в таблице (URL: /admin/agents?status=error). Повторный клик — сбрасывает. | P1 |
