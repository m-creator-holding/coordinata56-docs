## Принятые решения

| # | Точка выбора | Выбрано | Обоснование |
|---|---|---|---|
| H-P1 | Силуэт типа дома | Квадрат 32px с буквой типа (A/B/C/D) внутри, цвет по типу | Figma/SVG силуэты — отдельная фаза визуального дизайна (M-OS-1.4). Буква в квадрате — читаемо и однозначно. |
| H-P2 | Фильтр по типам | ToggleGroup с 5 кнопками: «Все / A / B / C / D» | shadcn/ui ToggleGroup — компактно, без выпадения. Позволяет мульти-выбор (A+B одновременно). |
| H-P3 | Фильтр по опциям | Combobox с чекбоксами внутри (мультиселект) | Опций может быть 10-20 (гараж, баня, терраса и т.д.); Combobox с поиском оптимален. |
| H-P4 | Grid view (плитки) | Карточка 200px: [тип-квадрат] Номер дома / Badge стадии / Прораб / Дата сдачи | Плитка даёт визуальный обзор; таблица — для работы с данными. Toggle в header переключает режим. |
| H-P5 | Карточка дома — количество tabs | 5 вкладок: Обзор / Опции / Платежи / Документы / История | Соответствует брифу. Аналогично компании в Admin UI (D10). |
| H-P6 | Доступ Foreman к вкладке Платежи | Tab виден, при клике — баннер «Нет доступа» | Tab скрывать нельзя — foreman должен понимать структуру карточки. Баннер объясняет ограничение. |
| H-P7 | Смена стадии | Кнопка «Перевести в стадию:» + DropdownMenu со следующими возможными стадиями | Прямая кнопка «следующая стадия» не учитывает возможный возврат (баг обнаружен → Фундамент). DropdownMenu даёт гибкость. Confirmation Dialog обязателен. |
| H-P8 | Монтаж опции (foreman) | Checkbox + дата установки inline в строке таблицы | Прораб на планшете: минимум кликов. Checkbox сразу видно что сделано/не сделано. |
| H-P9 | Колонка Бюджет в таблице домов | Скрыта для foreman через conditional render, не через CSS hidden | CSS hidden — data остаётся в DOM. Conditional render — не передаём данные на клиент вообще (API не возвращает financial поля для foreman role). |
| H-P10 | Пагинация списка домов | 85 домов — без пагинации, один список с виртуализацией | 85 строк не требуют страниц; виртуализация (react-window) рекомендована frontend для производительности. |

## Навигационная карта

```
/projects/{projectId}/houses             ← Страница 2A (этот файл)
  ├── клик на строку/плитку              → /houses/{houseId}
  └── breadcrumb «← Координата 56»      → /projects/{projectId}/overview

/houses/{houseId}                        ← Страница 2B (этот файл)
  ├── breadcrumb «← Дома»               → /projects/{projectId}/houses
  ├── вкладка Платежи (owner/dir/acc)   → inline список платежей
  └── вкладка История                   → inline timeline
```

## Экран 2A. Houses List — `/projects/{projectId}/houses`

### Persona и сценарий

**Персона A — Прораб (foreman) на планшете:**
Утром открывает список домов. Быстро фильтрует по типу «B» (свои объекты). Видит стадии, просроченные — выделены. Тапает на нужный дом, переходит в карточку.

**Персона B — Директор (director) на десктопе:**
Открывает список, переключает на Grid view для визуального обзора. Фильтрует по опции «Баня» — смотрит сколько домов с баней ещё не в стадии «Инженерия». Принимает решение.

## Экран 2B. House Card — `/houses/{houseId}`

**Персона A — Прораб (foreman):**
Тапает «Дом №14» из списка. Видит карточку: тип A, стадия «Фундамент», ответственный — он сам. Переходит на вкладку «Опции», отмечает checkbox «Баня — установлена». Нажимает «Перевести в стадию → Стены». Видит confirmation Dialog. Подтверждает.

Открывает карточку дома по ссылке из отчёта. Видит вкладку «Обзор» (readonly). Переходит на «Платежи» — видит список платежей этого дома. Экспортирует.
