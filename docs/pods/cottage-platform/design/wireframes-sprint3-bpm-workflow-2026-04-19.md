## Принятые решения

| # | Точка выбора | Выбрано | Обоснование |
|---|---|---|---|
| B1 | Редактор шаблона: структурная форма (A) vs JSON-редактор (B) | Вариант A — структурная форма с секцией шагов | Владелец не технический пользователь; форма снижает ошибки конфигурации; JSON-редактор — fallback для продвинутых пользователей в M-OS-2 |
| B2 | Шаги процесса: inline-редактирование vs Sheet | Inline, раскрывающийся Accordion на каждый шаг | Sheet перекрывает контекст; inline позволяет видеть все шаги сразу |
| B3 | История версий: вкладка vs правая панель | Правая панель (sticky scroll) по аналогии с /admin/rules | Консистентность с паттерном Admin UI; не требует переключения контекста |
| B4 | Placeholder «активных экземпляров»: отдельный роут vs колонка в списке | Колонка «Экземпляры» в таблице + отдельный роут-заглушка | Frontend не переделывает таблицу при запуске движка в 1.3; роут позволяет приготовить навигацию |
| B5 | Drag-to-reorder шагов | Описание UX-паттерна (handle-icon + aria-label), не код | Реализация на frontend через @dnd-kit; designer описывает только намерение |

## Role Matrix

| Роль | Доступ к /admin/bpm | Ограничения |
|---|---|---|
| owner | Полный CRUD шаблонов, шагов, статусов. Откат версий. | — |
| director | Просмотр + редактирование статуса существующих шаблонов | Не может создавать новые шаблоны или менять тип объекта у системных |
| accountant | Только просмотр (readonly). Баннер «Режим просмотра». | Кнопки создания и редактирования — display:none |
| foreman | 403 Forbidden — доступ запрещён | Прорабу конфигуратор BPM не нужен |

## Экран 1. `/admin/bpm/definitions` — Список шаблонов BPM

### Persona и сценарий

**Персона A — Владелец (owner):**
Открывает раздел «Шаблоны BPM» для проверки, что системные шаблоны «Согласование платежа» и «Приёмка работ» активны перед запуском M-OS-1.3. Фильтрует по статусу «Черновик» для поиска недоделанных шаблонов.

**Персона B — Директор (director):**
Хочет убедиться, что шаблон «Создание договора» задействован. Просматривает список без права создания.

**Персона C — Бухгалтер (accountant):**
Случайно попадает на страницу через sidebar. Видит баннер readonly. Данные видит — кнопки создания скрыты.

## Acceptance Criteria

| # | Критерий | Приоритет |
