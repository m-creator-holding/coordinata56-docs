## Суть замечания аудитора

> RFC-003 требует no-outbound и проверяет только mem0.ai / api.mem0.ai, но допускает embedding через Anthropic API. Это оставляет легальный сетевой обход за пределами tcpdump-критерия и ст. 45а CODE_OF_LAWS.

---

## Оценка: замечание корректно

Аудитор прав по существу. В исходном RFC §9 (примечание) было написано:

> «...либо через уже имеющийся Anthropic-клиент — зависит от финального выбора в пилоте».

Это формировало допустимый путь: `mem0 → embedder: anthropic → api.anthropic.com/v1/embeddings`. Такой вызов:
1. Передаёт текст воспоминаний (содержимое памяти субагентов, факты проекта, решения Владельца) на внешний сервер — нарушение принципа «данные проекта не уходят во внешние сервисы».
2. Не попадал в tcpdump-критерий (проверялись только `mem0.ai` / `api.mem0.ai`).
3. Формально нарушает ст. 45а CODE_OF_LAWS v2.1: «живые внешние интеграции — реальные сетевые вызовы к сторонним системам — запрещены до прохождения production-gate».

Таким образом: no-outbound-декларация RFC была неполной, допуская обход через embedding-канал.

## Выполненные правки (RFC-003 Amendment 2026-04-17)

### 1. Запрет Anthropic API для embedding (A1)

В RFC зафиксировано явно:
- Anthropic API `/v1/embeddings` — запрещён.
- Нормативное основание: ст. 45а CODE_OF_LAWS v2.1 + Решение 12 Владельца msg 1147.
- Единственный разрешённый embedding — `sentence-transformers` self-hosted.

### 2. Выбор конкретной embedding-модели для русского текста (A2)

Модель: `paraphrase-multilingual-mpnet-base-v2`.
- Размер: ~1.1 GB (fp32) на диске.
- Скачивается один раз при установке; runtime — полностью локальный.
- Поддерживает 50+ языков включая русский технический текст.
- Лицензия Apache-2.0.
- Альтернатива при ограничениях места: `paraphrase-multilingual-MiniLM-L12-v2` (~470 MB, чуть ниже качество).

### 3. Расширенный tcpdump-критерий (A3)

Предыдущий критерий закрывал только `mem0.ai / api.mem0.ai`. Новый охватывает все потенциальные каналы утечки:
- `api.anthropic.com` — embedding через Anthropic
- `api.openai.com` — если openai-пакет инициируется с дефолтным endpoint
- `api.huggingface.co` — если sentence-transformers проверяет обновление модели в runtime

Разрешённый egress в runtime: только localhost (ChromaDB), внутренняя сеть сервера. LLM-вызовы субагентов через `api.anthropic.com` остаются рабочим режимом Claude Code — не блокируются, но embedding-конфиг Mem0 не должен использовать этот хост.

### 4. Зависимости: sentence-transformers — обязательная (A4)

Статус изменён с «опциональной» на **обязательную**. Openai-пакет (транзитивная зависимость mem0ai) — настроить с dummy endpoint (`OPENAI_BASE_URL=http://localhost:9999`) до инициализации mem0.

### 5. Дополнительный шаг пилота (A5)

Добавлен Шаг 3а: проверка no-outbound sentence-transformers в runtime (strace или tcpdump по 10 embedding-запросам). Если обнаружен outbound к `api.huggingface.co` — инцидент по ст. 45б, доклад Владельцу. Митигация: `TRANSFORMERS_OFFLINE=1`.

## Что НЕ изменилось

- ChromaDB как векторное хранилище — не затронуто.
- Принцип записи только через Координатора-транспорт — не затронуто.
- Запрет Mem0 Cloud — не затронуто (был в исходном RFC).
- Метрики пилота §7.1 и §7.3 — не затронуты.
