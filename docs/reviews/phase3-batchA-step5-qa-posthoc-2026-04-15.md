## 6. Замечания

### minor — отсутствие аннотации возвращаемого типа на `db_session` и `client`

**Файл**: `test_batch_a_coverage.py`, строки 45, 54  
**Описание**: `async def db_session():` и `async def client(db_session: AsyncSession):` — без `-> AsyncGenerator[...]`. Большинство существующих test_*.py также без аннотации (кроме test_projects.py и test_auth.py), поэтому это проблема проекта в целом, не специфичная для нового файла.  
**Приоритет**: `minor` (консистентность с test_projects.py нарушена, но не критично).

### nit — импорт внутри тела тестового метода

**Файл**: `test_batch_a_coverage.py`, строка 1235  
```python
from app.models.house import HouseStageHistory
```
Импорт делается внутри метода `test_stage_change_audit_and_history_both_created`, хотя все остальные модели импортированы на уровне модуля (строки 24–30). Необходимо перенести в заголовочный блок импортов.  
**Приоритет**: `nit`

### nit — непоследовательная проверка `error.code` в RBAC-тестах

Часть тестов проверяет `resp.json()["error"]["code"] == "PERMISSION_DENIED"`, часть ограничивается только `assert resp.status_code == 403` (например, строки 242, 279, 290, 300, 346, 363, 377, 386, 453, 476, 486, 496). По DoD мини-батча A требуется проверка полного формата ADR 0005. Половина RBAC-тестов проверяет только код статуса, что достаточно для функционального покрытия, но не покрывает формат ответа.  
**Приоритет**: `nit` (формат ADR 0005 покрыт отдельным классом `TestErrorFormatADR0005`).
