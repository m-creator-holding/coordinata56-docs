## ADR-0014 — Anti-Corruption Layer

### Статус
`proposed` — ожидает governance. **P0-блокер Gate-0.**

### Полнота документа: УДОВЛЕТВОРИТЕЛЬНАЯ, но с двумя блокирующими пробелами

| Критерий | Оценка | Примечание |
|----------|--------|------------|
| Проблема и контекст | ЕСТЬ | Три риска с конкретной механикой, ссылки на CODE_OF_LAWS ст. 45а/45б |
| Решение с примерами | ЕСТЬ | Три состояния адаптера, guard-логика (5 шагов), интерфейс базового класса, conftest-пример |
| Альтернативы (≥2) | ЕСТЬ | B — feature flags, C — service mesh, оба с явным «Отклонён» |
| Последствия (trade-offs) | ЕСТЬ | Три раздела, включая зависимость от infra-director |
| Ссылки в m-os-1-plan.md | ЧАСТИЧНО | Аналогично ADR-0013: упомянут как кандидат, без ссылки на принятый документ |

### Правки, необходимые перед ratification

1. **Предусловие в DoD**: пункт «Seed-миграция создаёт 7 записей в `integration_catalog`» зависит от таблицы, схема которой определяется в ADR-0015 (не существует). Добавить явное предусловие: «ADR-0015 принят до реализации seed-миграции». Без этого DoD содержит неисполнимый пункт. (Зафиксировано в adr-consistency-audit C-11.)

2. **Amendment к перечню action-значений**: AuditLog в ADR 0007 не содержит `adapter_call_blocked` в enum `audit_log.action`. ADR-0014 вводит это значение, но не документирует расширение явно — нет ссылки на ADR 0001 §6 и не указано, что добавление значения разрешено правилом ADR-0013 (расширение enum). Добавить в раздел «Влияние на существующие ADR» явную запись: «ADR 0001 §6 — enum `audit_log.action` расширяется значением `adapter_call_blocked` (миграция по правилам ADR-0013, расширение enum разрешено без двухшагового процесса)». (Зафиксировано в adr-consistency-audit C-05.)

3. **Уточнение iptables-зависимости как non-blocking для Gate-0**: раздел «Открытый вопрос: egress-блокировка через iptables» правильно разграничивает ответственность, но не фиксирует явно, что DoD каркаса (пункты 1–13) не зависит от iptables. Добавить одну строку: «Пункты DoD 1–13 не блокируются решением infra-director; iptables является предусловием только для production-gate, не для Gate-0.»

### Кого опросить перед ratification

- **governance-director** — формальное утверждение (статус `proposed → accepted`); проверить соответствие CODE_OF_LAWS ст. 45а/45б
- **backend-director** — подтверждение, что guard-логика в разделе «Решение» технически реализуема в рамках текущей архитектуры FastAPI; уточнить механизм кеша TTL 60 сек при отсутствии ADR-0015
