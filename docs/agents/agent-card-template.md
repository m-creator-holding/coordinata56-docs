## Правила заполнения

1. **Все поля обязательны.** Если данных нет — явно писать «unknown», а не оставлять пустым. Это помогает аудитору Governance видеть пробелы.
2. **Ссылки на регламенты проверяем** перед коммитом — `git grep -l "<путь>"` или `test -f <путь>`.
3. **Карточка — источник истины для `agents-map.yaml`.** При изменении карточки — обновить yaml в том же коммите.
4. **Создание новой роли** — через заявку в `docs/governance/requests/YYYY-MM-DD-new-role-<slug>.md` (комиссия Governance).
5. **Уход в dormant / возвращение в active** — через комиссию + обновление статуса в yaml и в этом шаблоне.
