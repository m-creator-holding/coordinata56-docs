# Регламенты по направлениям (departments)

Здесь живут департаментские регламенты — внутренние правила работы каждого из шести направлений компании. Регламент пишет и владеет им Директор направления; согласует Начальник отдела; утверждает Координатор.

## Процесс обновления

Полная процедура — `docs/agents/regulations_addendum_v1.5.md` §3.2.

Кратко:
1. Сотрудники работают → Начальник видит ошибки
2. Начальник копит наблюдения → передаёт Директору
3. Директор пишет/обновляет регламент → согласует с Начальником
4. Согласованный регламент → Координатору на утверждение
5. После утверждения → автоматически читается всеми Сотрудниками этого направления перед каждой задачей

## Лимит

Каждый регламент — ≤200 строк. При росте — Директор обязан рефакторить (см. v1.5 §4.3).
