## Цель отдела

Непрерывный сенсинг внешних источников → анализ → пилот → внедрение. Отдел отвечает за то, чтобы наш инструментарий (скилы, субагенты, MCP-серверы, процессы) не отставал от того, что появляется в индустрии. Без этого отдела Координатор сам читает HN/GitHub/твиттер между делом — это непредсказуемо и часто упускается.

## Структура

| Уровень | Роль | Модель | Основная задача |
|---|---|---|---|
| L2 | `ri-director` | Opus | Стратегия, приоритизация находок, решения «пилот / отклонить», weekly digest Владельцу. |
| L3 | `ri-scout` | Sonnet | Ежедневное сканирование источников, triage, формирование очереди для анализа. |
| L3 | `ri-analyst` | Opus | Глубокий анализ находок scout'а: что делает, какую задачу решает, что у нас уже есть, дельта, риски, оценка внедрения. Пишет RFC. |

Пилот-раннер отсутствует в MVP — пилоты по коду гоняет `backend-dev` или `qa` из соответствующих отделов по запросу ri-director через Координатора.

## Обязательная расшифровка простым языком (добавлено 2026-04-15 по указанию Владельца)

Каждая находка Scout'а и каждый RFC Analyst'а **обязательно** содержат три пояснения на обычном русском (без англицизмов и технических терминов без расшифровки), ориентированные на читателя-нетехника (Владелец):
1. **«Что это делает»** — одним абзацем, как если бы Вы объясняли это строителю или бухгалтеру. Аналогия приветствуется.
2. **«Чем это нам полезно»** — конкретный сценарий внутри coordinata56, а не общие слова.
3. **«Что именно предлагаем внедрить»** — шаги внедрения, кто участвует, срок, стоимость.

Scout добавляет пункты 1–2 в findings.md (колонка «Одной строкой» расширяется до мини-абзаца в примечании ниже строки, либо отдельный файл `docs/research/briefs/<slug>.md`). Analyst в RFC раскрывает все три пункта в начале документа перед техническим анализом.

## Источники для сканирования (стартовый набор)

1. GitHub Trending weekly, теги: `ai-agents`, `llm`, `mcp`, `claude-code`, `prompt-engineering`.
2. Anthropic Changelog / docs (`docs.anthropic.com/en/release-notes`).
3. `awesome-claude-code`, `awesome-mcp-servers` на GitHub.
4. Hacker News front page с фильтром по AI/dev tools.
5. Блог Anthropic (anthropic.com/news).
6. Simon Willison's Weblog (simonwillison.net).

Список расширяется по предложению Scout'а через апруваемую заявку Директору.

## Совмещение ролей при малом потоке (добавлено 2026-04-15 по итогам пилота)

При среднем потоке <3 находок в неделю Scout и Analyst могут совмещаться в одном субагенте-исполнителе. Разделение становится обязательным, когда поток стабильно >5 находок/неделю две недели подряд.

## Weekly digest (уточнено 2026-04-15)

Каждый понедельник 10:00 МСК ri-director через Координатора отправляет Владельцу в Telegram:
- 3–7 находок с расшифровкой по трём пунктам (см. раздел «Обязательная расшифровка»)
- Что в пилоте + метрики «до/после»
- Что внедрили / отклонили за прошлую неделю

Формат — обычный текст ≤600 слов, без markdown-разметки сложнее жирного и списков.

## Процесс (Kanban на 4 колонки)

1. **Discovered** — Scout нашёл, строка в `docs/research/findings.md` с датой, ссылкой, 1-строчным описанием.
2. **Analysis** — Analyst берёт в работу, пишет RFC в `docs/research/rfc/rfc-NNN-<slug>.md` (шаблон: проблема → решение → аналог у нас → дельта → риск → оценка внедрения).
3. **Pilot** — Директор решил попробовать, запрашивает Координатора на выделение dev/qa для пилота на изолированной задаче (worktree). Метрики фиксируются в RFC.
4. **Adopted / Rejected** — финальное решение Директора (если тактика) или Координатора→Владельца (если стратегия или стоит денег).

## Кросс-вертикальная коммуникация

- Директор может общаться с architect, backend-director, quality-director напрямую — узнавать, есть ли у них уже похожее, что им не хватает.
- Scout и Analyst между департаментами напрямую НЕ общаются — через Директора.

## Бюджет внимания (жёсткий)

- Scout: до 2 часов в день (сессия до 4 вызовов `WebFetch` + `WebSearch`).
- Analyst: до 4 часов на один RFC. Не уложился — эскалация Директору с коротким статусом.
- Pilot: не более 2 рабочих дней. Не уложился — статус «нежизнеспособно в текущем виде», отклонение или правка скоупа.

## Weekly digest

Каждый понедельник 10:00 (фикс до отмены Владельцем) ri-director через Координатора отправляет Владельцу краткий дайджест:
- 3-7 находок с коротким пояснением.
- Что в пилоте и метрики.
- Что внедрили / отклонили за прошлую неделю.

## Что НЕ делает отдел

- Не пишет производственный код (пилоты запускает через backend-dev/qa).
- Не коммитит в git (коммит — Координатор).
- Не меняет регламенты других департаментов (это governance-director).
- Не общается с Владельцем напрямую (через Координатора).

## Пилотный запуск

Пилот 2026-04-15: ri-scout сканирует GitHub Trending и Anthropic release notes за период 2026-04-01..2026-04-15, ri-analyst разбирает топ-3 находки, ri-director готовит отчёт Координатору. Оцениваем качество находок и решаем — масштабируемся или дорабатываем регламент.
