## Сферы ответственности отдела (предварительно)

- Docker, Docker Compose локального окружения
- CI/CD пайплайны (когда дойдём до Фазы 9)
- PostgreSQL: схема, миграции, индексы, performance
- Бэкапы БД
- Деплой на staging и production

- Интеграции с внешними системами (банки, ОФД, Дадата, Росреестр, 1С)
- Anti-corruption layer для интеграций

## Правила (по уже накопленному опыту)

1. **Миграции Alembic — обязательный round-trip**: `downgrade -1 && upgrade head` чисто, без ошибок. Без round-trip миграция не считается готовой. *(Уже в CLAUDE.md.)*
2. **Enum в миграции — значения совпадают с `.value` Python-enum** (регистр). *(Поймано: Phase 3 Batch A step 1.)*
3. **Никаких боевых данных без явного согласования Владельца** (regulation v1.0).
4. **Production-операции (`docker compose down`, `migrate prod`, `restart`) — только с явного разрешения** Владельца.
