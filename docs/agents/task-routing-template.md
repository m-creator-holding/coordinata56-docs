1. Координатор → {{ Директор }} — «сделай X, срок Y»
2. {{ Директор }} → {{ Head }} — декомпозиция на шаги 1..N
3. {{ Head }} → {{ Workers }} — распределение по исполнителям с FILES_ALLOWED
4. {{ Workers }} → self-check → {{ Head }}
5. {{ Head }} → первичное ревью → {{ Директор }}
6. {{ Директор }} → финальное ревью → Координатор
7. Координатор → `reviewer` (staged diff) → вердикт approve
8. Координатор → коммит
9. Координатор → отчёт Владельцу в Telegram
10. Координатор → запись в `project_tasks_log.md` + обновление статус-файла фазы

## Правила применения шаблона

1. **Шаблон заполняется до начала работы**, не в ретроспективе. Без паспорта — задача не стартует (исключение — XS срочные, с пост-фактум заполнением в тот же день).
2. **Изменение плана во время работы** — фиксируется в том же документе в разделе «История изменений», а не переписывается молча.
3. **Статус задачи** — см. схему «Д» в `agents-diagrams.md`. Обновляется Координатором.
4. **При завершении** — паспорт прикрепляется к ретроспективе (если крупный) или сразу в `project_tasks_log.md` одной строкой (для мелких).
