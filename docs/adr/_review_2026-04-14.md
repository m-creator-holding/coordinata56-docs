# Ревью ADR-0001 и ADR-0002

**Дата**: 2026-04-14
**Reviewer**: субагент `reviewer`

## ADR-0001 «Модель данных v1» — `request-changes`

Блокирующие для Фазы 1 (не для Фазы 0):

- **[major] Нет секции «Проблема»** — смешана с Контекстом. Регламент требует явное разделение.
- **[major] Нет ER-диаграммы / списка FK с кардинальностью.** 16 сущностей перечислены, но связи не зафиксированы — разные субагенты будут трактовать схему по-разному.
- **[major] Нет стратегии хранения паролей** для `users` (bcrypt/argon2id? колонка `password_hash`?).
- **[major] Отсутствует таблица `clients`/`buyers`** при заявленном «полном цикле под ключ». Либо добавить, либо явно зафиксировать out-of-scope MVP1.
- **[minor] Soft-delete**: риск «забыть фильтр» отмечен, но митигация (global scope / RLS) не описана.
- **[minor] `house_configurations.price_cents`** — неясна семантика: цена опции или суммарная цена.
- **[minor] `budget_plan`** — неясна гранулярность строки (проект/стадия/дом/статья одновременно?).

## ADR-0002 «Выбор стека» — `approve` с условиями

- **[major] Нет решения по State Management** на фронте. Зафиксировать Zustand (или альтернативу) явно.
- **[minor] `python-jose`** заброшен с 2022, есть CVE. Заменить на `PyJWT` или `authlib`.
- **[minor] Версии** некоторых пакетов «актуальный на момент инициализации» — нодетерминизм. Зафиксировать конкретные версии.
- **[minor] HTTP-клиент**: «axios или fetch» — выбрать один.
- **[minor] Tailwind v3 vs shadcn/ui ≥ 2.0 (требует v4)** — синхронизировать версии.

## Согласованность ADR-0001 и ADR-0002

Противоречий нет. ADR-0002 корректно ссылается на ADR-0001. Gap: RBAC на фронте не описан; механизм записи в `audit_log.changes_json` не зафиксирован — отложить в ADR-0003 «Layered architecture & audit».

## Готовность к Фазе 0

Блокирующих замечаний нет. Фазу 0 можно начинать. Фазу 1 — после устранения major-замечаний ADR-0001.
