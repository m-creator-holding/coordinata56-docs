# coordinata56 — публичное зеркало документации

Это **read-only зеркало** папки `docs/` внутреннего проекта `coordinata56`.

## Зачем

Зеркало создано для того, чтобы внешние ИИ-консультанты (например, другие LLM) и приглашённые эксперты могли читать документацию проекта без доступа к основному приватному репозиторию с исходным кодом.

## Что внутри

- `docs/adr/` — архитектурные решения (ADR)
- `docs/agents/` — регламенты системы ИИ-субагентов (CODE_OF_LAWS, правила делегации, event log schema, карточки агентов, диаграммы)
- `docs/governance/` — заявки на правки регламента, аудиты, changelog
- `docs/research/` — R&I pipeline: findings, RFC, пилоты
- `docs/reviews/` — отчёты код-ревью по фазам
- `docs/phases/` — скоуп, решения, tech-debt по фазам
- `docs/stories/` — User Stories
- `docs/specs/` — технические спецификации
- `docs/knowledge/` — ретроспективы, глоссарий, KPI-каталог, lessons learned
- `docs/qa/` — тест-стратегия, отчёты QA
- `docs/security/` — threat-model и security-отчёты
- `docs/wireframes/` — UX-wireframes MVP
- `docs/legal/` — юридические чек-листы (152-ФЗ и др.)
- `docs/integrations/` — карта внешних систем
- `README.md`, `CLAUDE.md` — корневые документы проекта (правила, антипаттерник)

## Чего нет в зеркале

- Исходный код (`backend/`, `frontend/`, `dashboard/`, `migrations/`) — остаётся в приватном репо.
- Секреты, токены, пароли — не хранятся в `docs/` и не переносятся.
- Личные идентификаторы Владельца (Telegram chat_id) заменены на заглушку `<OWNER_CHAT_ID>`.

## Синхронизация

Зеркало обновляется **вручную**, на момент публикации — snapshot от 2026-04-16.

Актуальность конкретного документа сверяйте по полю даты внутри него (обычно указано в шапке ADR, аудитов, отчётов).

## Лицензия и использование

Документы опубликованы для консультаций и обсуждений. Коммерческое переиспользование регламентов и ADR — по согласованию с Владельцем проекта.
